### 双远程仓库策略

| Remote | URL | 用途 | 推送内容 |
|--------|-----|------|---------|
| **origin** | `git@github.com:zhaog100/openclaw-skills.git` | 公共技能 | `skills/`, `SKILL.md`, 技能脚本 |
| **xiaomijiao** | `git@github.com:zhaog100/xiaomijiao-skills.git` | 个人工作区 | `memory/`, `docs/`, `intel/`, `MEMORY.md` |

### 分支管理规则

1. **主分支**: `main` 或 `master` (统一为 `main`)
2. **功能分支**: `feat/xxx` (短期开发)
3. **Bounty 分支**: `bounty-#N` (完成后立即合并删除)
4. **发布分支**: `release/vX.Y.Z` (版本发布)

### 清理计划

```bash
# 删除已完成的 bounty 分支
git branch -d bounty-1501
git branch -d solfoundry-sync

# 统一主分支名称
git branch -m master main
git push origin -u main
```

## 📚 QMD 索引重建

### 问题诊断
- 当前状态: "没有 collection"
- 根因: QMD 数据库为空，需重新索引

### 重建步骤

```bash
# 1. 检查 QMD 配置
qmd config show

# 2. 创建 collection
qmd collection create xiaomila-knowledge

# 3. 添加索引路径
qmd index add /home/zhaog/.openclaw-xiaomila/workspace/knowledge
qmd index add /home/zhaog/.openclaw-xiaomila/workspace/memory
qmd index add /home/zhaog/.openclaw-xiaomila/workspace/docs

# 4. 执行索引
qmd index run --all

# 5. 验证索引
qmd search "bounty 狩猎"
```

在 crontab 中已有:
