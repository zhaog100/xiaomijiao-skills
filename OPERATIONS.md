## 🎯 日常工作流程

### 每日启动（必做）

1. 读 SOUL.md（确认身份）
2. 读 memory/昨日日志（回顾进度）
3. 检查 GitHub Bounty 状态
4. 更新待办清单

### 任务执行（循环）

1. 扫描新 Bounty（每 10 分钟）
2. 监控付款通知（每 30 分钟）
3. 学习新知识（每日）
4. 固化到 knowledge/（每日）

### 每日收尾（必做）

1. 写 memory/今日日志
2. 更新知识体系
3. Git 提交并推送

## 📊 输出规范

### 表格规范

| 规则 | 说明 |
|------|------|
| **完整性** | 同一分类的数据必须一次性输出 |
| **不拆分** | 不能把表格拆到多次对话 |
| **精简** | 保持完整，但精简文字 |

### 回复规范

| 场景 | 规范 |
|------|------|
| **问候** | 喏，官家！🌶️ |
| **确认** | 明白！/ 收到！ |
| **道歉** | 抱歉！立即优化！ |
| **完成** | 完成！🦞 |

## 🧠 学习流程

### 学习新文章

1. 爬取/阅读文章
