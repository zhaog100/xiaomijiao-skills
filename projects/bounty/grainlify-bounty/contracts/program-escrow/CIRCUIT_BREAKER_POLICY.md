# Circuit Breaker Policy - Program Escrow

This document outlines the circuit breaker implementation for the `program-escrow` contract, designed to protect funds and provide full auditability of emergency actions.

## Overview

The circuit breaker is a safety mechanism that monitors operational health and automatically pauses critical functions (like payouts) if predefined error thresholds are exceeded. It also provides administrative controls for manual intervention.

## Operational States

| State | Description | Payouts Allowed | Transition To |
|-------|-------------|-----------------|---------------|
| **Closed** | Normal operation. | Yes | **Open** (on failure threshold breach) |
| **Open** | Emergency pause active. | No | **HalfOpen** (after cooldown or admin reset) |
| **HalfOpen** | Trial period after a pause. | Yes (limited) | **Closed** (on success) or **Open** (on failure) |

## Audit Events

For off-chain monitoring, the following events are emitted for all circuit breaker actions:

### State Transitions
- `cb_open`: Emitted when the circuit opens. Includes failure count and reason (`auto` or `manual`).
- `cb_half`: Emitted when the circuit reaches HalfOpen state.
- `cb_close`: Emitted when the circuit returns to Closed state.

### Administrative Actions
- `cb_adm`: Emitted when the circuit admin is changed. Includes old and new admin addresses.
- `cb_reset`: Emitted when an admin manually resets the circuit breaker.
- `cb_cfg`: Emitted when the circuit breaker configuration is updated.
- `rate_lim`: Emitted when rate limiting parameters are updated.

### Operational Audits
- `cb_fail`: Emitted on operation failure. Includes failure count, operation name, and error code.
- `cb_success`: Emitted on successful operations to track recovery.

## Configuration

The circuit breaker can be configured with the following parameters:
- **Failure Threshold**: Number of consecutive failures before opening.
- **Success Threshold**: Number of consecutive successes in HalfOpen before closing.
- **Cooldown Period**: Minimum time the circuit remains Open before it can be reset.

## Emergency Actions

Administrators can perform the following emergency actions:
1. `set_circuit_admin`: Rotate the authority for circuit breaker management.
2. `reset_circuit_breaker`: Manually move the circuit from Open to HalfOpen.
3. `configure_circuit_breaker`: Adjust thresholds based on current network conditions.
4. `emergency_withdraw`: Withdraw all funds from a paused program (requires contract admin).
