# 旅行客平台项目需求说明

> **一句话总结**：管理端主导规则，平台端线路查看及订单下发、司机端接单打卡、用户端查看行程

---

## 📋 项目概述

**核心定位：** 旅游用车综合服务平台

**四端设计：**
- **管理端**：规则配置、订单管理、用户管理、统计看板
- **旅行社端（Web）**：线路查看、订单下发、结算管理
- **司机端（移动）**：接单、打卡、紧急求助
- **用户端**：查看行程

**技术栈：** Vben Admin + Vue3 + Vite（SPA架构）

---

## 🏗️ 系统架构

### 核心业务流程

```
需求创建 → 价格试算 → 定价审批 → 订单拆分 → 派单 → 异常处理 → 行程执行 → 完成
```

### 价格三层模型

```
最上层：规则分类（管理和权限）
    ↓
中间层：价格规则（组合和计算）
    ↓
最底层：费用因子（定义费用本身）
```

**设计原则：** 新增客户、新增费用，都不会影响已有规则

---

## 🎯 四大端功能模块

### 1️⃣ 管理端

#### 用户管理
- **司导用户管理**
  - 注册 → 人工审核 → 星级评定（1-5星+超五星）
  - 审核状态：待审核/已通过/已驳回
  - 评分维度：服务态度、驾驶技术、服务规范

- **旅行社用户管理**
  - 四步注册流程（基本信息 → 资质材料 → 账号密码 → 审核通知）
  - 审核周期：1-3个工作日
  - 多端登录限制：最多2台设备

#### 规则配置
- **价格规则管理**
  - 规则命中逻辑：必须命中且仅命中1条规则
  - 优先级：系统根据条件复杂度自动生成
  
- **费用管理**
  - 费用因子：车辆基础费、司机基础费、导游费、补贴类费用
  - 参数配置：主体/城市/星级/季节
  
- **时间配置**
  - 旺季时间配置（其他时间默认淡季）
  - 对历史订单不生效

#### 业务管理
- **订单管理**
  - 订单状态：议价中/已定价待派单/派单中/已完成/已取消
  - 智能拆单：主订单 → N个子订单（按车辆/司机/行程天数）
  - 派单模式：系统轮询（15分钟）+ 指定司机（强派）
  
- **线路管理**
  - 线路信息维护
  - 司导信息查看
  
- **产品管理**
  - Tab分类：全部产品/下游产品/适配产品
  - 状态管理：已上架/已下架/审核中/已通过/草稿

#### 财务管理
- **旅行社结算**
  - 结算周期管理
  - 佣金比例：固定10%
  - 状态：待结算/已结算
  
- **司导报销**
  - 审批流程：待审批/已结算
  - 费用证明：需提供支付截图或照片

#### 应急处理
- **紧急求助管理**
  - 严重程度：紧急（红）/严重（橙）/一般（蓝）
  - 状态：待处理/已处理/已驳回
  
- **申诉处理**
  - 纠纷事实核查
  - 责任判定：司机责任/游客责任
  - 处理结果：通过/不通过

#### 统计看板
- 核心数据：订单总数/销售金额/司导人数/旅行社数量
- 订单状态分布（环形图）
- 异常订单列表
- 订单销售趋势（双轴折线柱状图）

### 2️⃣ 旅行社Web端

#### 首页看板
- 核心指标：订单数量/待处理订单/行程订单数/总交易额
- 订单趋势分析（双轴趋势图）
- 快捷操作：处理订单/导出订单/异常订单/接收订单

#### 订单管理
- **新建订单**
  - 基本信息：产品名称/线路名称/旅客人数/订单类型/开始时间/行程天数
  - 行程规划：多日行程结构（Day 1 / Day 2 / ...）
  - 服务等级：司导星级（1-5星+超五星）/车型/品牌
  - 计价模式：自动计价/手动议价
  
- **订单操作**
  - 查看详情/议价审批/取消/处理/完成标记

#### 结算管理
- 结算单列表
- 结算明细查看

---

### 3️⃣ 司机端（移动）

#### 首页
- 数据概览：本月收入/接单数/完成订单数
- 今日行程展示
- 待处理派单提醒
- 快捷入口：上传记录/附加费用/紧急求助/导航线路

#### 行程详情
- **Tab结构**
  - 行程安排：当日行程 + 快捷打卡
  - 行程记录：该线路所有行程安排
  - 照片上传：行程照片管理
  
- **定位打卡**
  - 针对每个任务点提供打卡按钮

#### 功能模块
- **照片上传**
  - 拍照上传/相册选择
  - 标签分类（如"实拍"）
  
- **附加费用**
  - 添加当前行程/历史行程的附加费用
  
- **紧急求助**
  - 进行中行程特殊情况上报
  
- **导航线路**
  - 默认当前已接单行程导航

---

### 4️⃣ 旅行社小程序

#### 功能边界
- ✅ 保留功能：新建订单/订单列表/订单详情/状态查看
- ❌ 不支持：订单拆分/派单（后台系统处理）

#### 首页结构
- 统计卡片：订单总数/销售金额/进行中/审批中
- 快捷入口：一键下单
- 订单列表（状态筛选）

#### 下单流程（2步）
1. **订单信息**
   - 基本信息：产品名称/线路名称/订单类型/订单人数/开始时间/行程天数
   - 行程详情：每日行程 + 备注
   - 车型与司导配置：司导要求/意向车型/意向品牌/客户姓名/客户电话
   
2. **计价提交**
   - 自动计价（默认）/ 手动议价
   - 订单信息确认
   - 确认下单

## 🔧 核心交互规则

### 登录注册
- **注册流程**
  - 基本信息 → 资质材料 → 账号密码 → 审核通知
  - 企业类型：旅行公司/租车公司
  - 必传材料：营业执照 + 专项许可证
  
- **登录方式**
  - 账号密码登录
  - 微信一键登录（需绑定）
  
- **密码重置**
  - 手机号 + 图形验证码 + 短信验证码 + 新密码

### 订单拆分规则
- **拆单维度**
  - 按行程天数拆分
  - 按车辆拆分
  - 按司机数量拆分
  
- **拆单结果**
  - 1个主订单 → N个子订单
  - 子订单独立维护：派单状态/司导信息/异常状态
  
- **状态联动**
  - 主订单状态 = 子订单状态聚合
  - 任一子订单失败 → 主订单进入异常处理中

### 派单与响应
- **派单模式**
  - 系统轮询（派单制）：15分钟，2小时未接进入异常
  - 指定司机（强派制）
  
- **响应时效**
  - 每个子订单单独计时
  - 任一子订单派单失败不影响其他子订单

### 价格计算
- **算价流程**
  ```
  选择线路 → 生成价格上下文 → 规则筛选（命中1条）
  → 获取费用因子集合 → 参数取值 → 逐项算价 → 价格汇总
  ```
  
- **PriceContext（价格上下文）**
  - 线路ID/天数/城市/季节/团型/星级
  - 角色标识：司机/导游/翻译
  
- **异常处理**
  - 命中失败：阻断算价
  - 多条命中：规则配置错误
  - 参数缺失：使用默认值

## 📊 状态机设计

### 订单状态流转
```
新建 → 议价中 → 已定价/待派单 → 派单中 → 执行中 → 已完成
                ↓
            异常处理中 → 已取消
```

### 用户状态
- **司导用户**：待审核 → 已通过 / 已驳回
- **旅行社用户**：待审核 → 已通过 / 已驳回

### 结算状态
- **旅行社结算**：待结算 → 已结算
- **司导报销**：待审批 → 已结算
