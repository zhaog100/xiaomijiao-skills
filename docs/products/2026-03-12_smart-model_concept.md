## 💡 产品愿景

**一句话描述**：让 AI 根据任务复杂度、文件类型、上下文使用量等因素，自动选择最优模型，提升响应质量和效率。

---

## 🎯 目标用户

**主要用户**：
- OpenClaw 用户（需要频繁使用 AI 模型）
- 多模型环境用户（有多个 API Key/模型可选）

**次要用户**：
- 开发者（需要针对不同任务使用不同模型）
- 企业用户（需要控制 API 成本）

---

## 📋 使用场景

### 场景 1：简单问答
- **用户行为**：问"今天天气如何？"
- **当前问题**：可能使用高成本模型（浪费）
- **smart-model 方案**：自动使用轻量模型（如 Flash）

### 场景 2：复杂代码开发
- **用户行为**：发送代码文件，请求重构
- **当前问题**：可能使用轻量模型（质量不够）
- **smart-model 方案**：自动使用强模型（如 GPT-4/Claude-3）

### 场景 3：长文档分析
- **用户行为**：上传 50KB 文档，请求总结
- **当前问题**：可能超出模型上下文限制
- **smart-model 方案**：自动选择大上下文模型（如 200K 上下文模型）

### 场景 4：图片内容识别
- **用户行为**：发送截图，问"这是什么？"
- **当前问题**：文本模型无法处理
- **smart-model 方案**：自动切换到多模态模型

## 🎯 核心功能（P0）

### 功能 1：文件类型检测 ⭐⭐⭐
- **描述**：自动检测用户发送的文件类型
- **输入**：文件路径/内容
- **输出**：文件类型（代码/文档/图片/无文件）
- **规则**：
  - 代码文件（.py/.js/.sh/.ts/.java 等）→ Coding 模型
  - 文档文件（.md/.txt/.pdf/.docx 等）→ 大上下文模型
  - 图片文件（.jpg/.png/.gif/.webp 等）→ Vision 模型
  - 无文件 → 根据消息复杂度判断

### 功能 2：多维度消息复杂度分析 ⭐⭐⭐⭐⭐
- **描述**：4 维度加权评分（参考 smart-model-switch v1.1.0）
- **输入**：用户消息文本
- **输出**：复杂度评分（0-10 分）
- **评分维度**：
  - **长度评分**（30%）：<50 字 (1 分) <200 字 (2 分) <500 字 (3 分)
  - **关键词评分**（40%）：简单 (0 分) 中等 (2 分) 复杂 (3 分)
  - **代码检测**（20%）：无代码 (0 分) 少量 (2 分) 大量 (3 分)
  - **视觉检测**（10%）：无视觉 (0 分) 有视觉 (3 分)
- **模型映射**：
  - 0-3 分 → Flash（轻量模型）
  - 4-6 分 → Main（标准模型）
  - 7-8 分 → Complex（强模型）
  - 9-10 分 → Complex + 深度分析模式

### 功能 3：上下文监控与保护 ⭐⭐⭐⭐⭐
- **描述**：监控当前会话的上下文使用量（参考 context-manager v2.2.2）
- **输入**：session_status 输出
- **输出**：上下文占用率 + 切换建议
- **规则**：
  - <60% → 当前模型继续 ✅
  - 60-85% → 准备切换，提醒用户 ⚠️
  - >85% → 强制切换到新会话 + 大上下文模型 🚨
- **防频繁切换机制**：
  - 冷却期：切换后 10 分钟内不再检测
  - 连续性要求：连续 2 次超过阈值才触发
  - 通知间隔：5 分钟内不重复提醒

### 功能 4：AI 主动检测机制 ⭐⭐⭐⭐⭐（核心创新）
- **描述**：每次 AI 回复前自动检查（参考 integrate-check.sh）
- **触发时机**：AI 回复前的钩子（hook）
- **执行逻辑**：
  1. 检查上下文使用率
  2. 分析消息复杂度
  3. 判断是否需要切换模型
  4. 输出决策（静默或提醒）
- **防骚扰机制**：
  - 静默模式：无问题时不输出
  - 冷却期：切换后 10 分钟内不检测
  - 智能提醒：友好格式，包含原因和建议

### 功能 5：自动切换执行 ⭐⭐⭐⭐
- **描述**：根据规则自动切换模型
- **输入**：模型切换决策
- **输出**：更新配置或发送系统事件
- **执行模式**：
  - **本地模式**：修改 agents.json（session-memory 集成）
  - **远程模式**：发送系统事件到 Gateway（cron 集成）
  - **混合模式**：优先本地，失败时远程
