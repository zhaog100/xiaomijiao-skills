## 1. 需求概述

### 1.1 背景

**官家原话**：
> "你现在这个需求：创建一个技能，实现定时（中午12点、晚上23：50分）仔细回顾今天的事和学习的知识，进行查漏补缺，并更新记忆、知识库、Git库，同时更新远程仓库信息。。以免单独策略会被遗忘。"

**核心问题**：
- ❌ 单独策略容易被遗忘
- ❌ 缺乏定时回顾机制
- ❌ 记忆、知识库、Git库更新不及时
- ❌ 查漏补缺依赖人工触发

### 1.2 目标

**核心目标**：自动定时回顾今日工作，查漏补缺，保持记忆和知识库同步更新

**具体目标**：
1. ✅ 每天中午12:00定时回顾上午工作
2. ✅ 每晚23:50定时回顾全天工作
3. ✅ 自动查漏补缺
4. ✅ 自动更新记忆（MEMORY.md + memory/YYYY-MM-DD.md）
5. ✅ 自动更新知识库（knowledge/）
6. ✅ 自动提交Git
7. ✅ 自动推送远程仓库

### 1.3 范围

**包含**：
- ✅ 定时回顾今日工作
- ✅ 学习知识总结
- ✅ 查漏补缺分析
- ✅ 记忆更新（MEMORY.md + memory/YYYY-MM-DD.md）
- ✅ 知识库更新（knowledge/）
- ✅ Git提交和推送

**不包含**：
- ❌ 跨日回顾（只回顾当天）
- ❌ 未来计划（只总结已完成）
- ❌ 外部系统集成

## 2. 功能需求

### 2.1 核心功能

#### 2.1.1 定时触发机制 ⭐⭐⭐⭐⭐

**触发时间**：
- ✅ **中午12:00** - 回顾上午工作（07:00-12:00）
- ✅ **晚上23:50** - 回顾全天工作（07:00-23:50）

**实现方式**：
- ✅ 使用Crontab定时任务
- ✅ 两次触发独立运行
- ✅ 避免重复执行（锁文件机制）

#### 2.1.2 回顾今日工作 ⭐⭐⭐⭐⭐

**回顾内容**：
1. ✅ 读取今日日志（memory/YYYY-MM-DD.md）
2. ✅ 读取MEMORY.md（长期记忆）
3. ✅ 读取Git提交记录
4. ✅ 读取GitHub Issues状态
5. ✅ 读取知识库更新

**分析维度**：
- ✅ 已完成任务
- ✅ 进行中任务
- ✅ 待办任务
- ✅ 遇到的问题
- ✅ 学到的知识
- ✅ 改进措施

#### 2.1.3 查漏补缺分析 ⭐⭐⭐⭐⭐

**检查项目**：
1. ✅ **记忆遗漏** - 是否有重要事件未记录到memory/YYYY-MM-DD.md
2. ✅ **知识遗漏** - 是否有新知识未更新到knowledge/
3. ✅ **Git遗漏** - 是否有未提交的重要文件
4. ✅ **远程遗漏** - 是否有未推送的本地提交

**自动识别**：
- ✅ 对比MEMORY.md和memory/YYYY-MM-DD.md
- ✅ 对比本地和远程Git状态
- ✅ 检查知识库文档数量

#### 2.1.4 更新记忆 ⭐⭐⭐⭐⭐

**更新内容**：
1. ✅ **memory/YYYY-MM-DD.md**
   - 补充遗漏事件
   - 更新任务状态
   - 添加学习总结
   - 记录改进措施

2. ✅ **MEMORY.md**
   - 提炼今日精华
   - 更新长期记忆
   - 添加新知识点

**更新策略**：
- ✅ 保留原有内容，追加新内容
- ✅ 不重复记录相同内容
- ✅ 保持时间顺序

#### 2.1.5 更新知识库 ⭐⭐⭐⭐⭐

**更新内容**：
- ✅ 新增知识文档（knowledge/YYYY-MM-DD/）
- ✅ 更新知识索引（knowledge/KNOWLEDGE-INDEX.md）
- ✅ 分类归档（按主题分类）

**知识来源**：
- ✅ 今日学习的知识
- ✅ 解决问题的方法
- ✅ 官家的教导
- ✅ 错误教训

#### 2.1.6 Git提交和推送 ⭐⭐⭐⭐⭐

**Git操作**：
1. ✅ **Git Add** - 添加所有更新文件
2. ✅ **Git Commit** - 提交更改
3. ✅ **Git Push** - 推送到远程仓库

**提交信息模板**：
```
chore(daily): YYYY-MM-DD 定时回顾更新

【回顾时间】12:00/23:50
【回顾范围】07:00-12:00/07:00-23:50

【更新内容】
- ✅ 记忆更新：memory/YYYY-MM-DD.md（+X行）
- ✅ 知识更新：knowledge/YYYY-MM-DD/（X个文档）
- ✅ Git提交：X个文件

【查漏补缺】
- ✅ 已补充：X项遗漏
- ✅ 已修复：X个问题
```

## 3. 技术要求

### 3.1 技术栈

| 组件 | 技术选型 | 理由 |
|------|---------|------|
| **定时任务** | Crontab | 系统自带，稳定可靠 |
| **脚本语言** | Bash + Python | Bash处理系统操作，Python处理数据分析 |
| **Git操作** | Git CLI | 官方工具，功能完整 |
| **文件处理** | Python | 字符串处理、文件读写方便 |
| **日志记录** | Log文件 | 简单直接，易于调试 |

### 3.2 目录结构

```
skills/daily-review-assistant/
├── SKILL.md                  # 技能说明
├── README.md                 # 使用文档
├── package.json              # 配置文件
├── install.sh                # 安装脚本
├── skill.sh                  # 主脚本
├── lib/
│   ├── reviewer.py          # 回顾分析器
│   ├── memory_updater.py    # 记忆更新器
│   ├── knowledge_updater.py # 知识库更新器
│   └── git_manager.py       # Git管理器
├── config/
│   └── config.yaml          # 配置文件
├── logs/
│   ├── review-YYYY-MM-DD.log    # 回顾日志
│   └── error-YYYY-MM-DD.log     # 错误日志
└── templates/
    ├── daily-summary.md      # 日报模板
    └── commit-message.txt    # 提交信息模板
```

### 3.3 配置文件

**config.yaml**：
```yaml
# 定时回顾更新助手配置

schedule:
  morning: "12:00"  # 中午12:00
  evening: "23:50"  # 晚上23:50

paths:
  workspace: "/root/.openclaw/workspace"
  memory: "memory/"
  knowledge: "knowledge/"
  logs: "skills/daily-review-assistant/logs/"

review:
  morning_range: "07:00-12:00"
  evening_range: "07:00-23:50"
  
update:
  memory: true
  knowledge: true
  git: true
  remote: true

git:
  commit_message_template: "templates/commit-message.txt"
  auto_push: true
```

## 6. 工作流程

### 6.1 中午回顾流程

```
1. Crontab 12:00触发
   ↓
2. 检查锁文件（避免重复执行）
   ↓
3. 读取memory/YYYY-MM-DD.md（07:00-12:00）
   ↓
4. 读取Git提交记录（07:00-12:00）
   ↓
5. 分析已完成任务
   ↓
6. 查漏补缺
   ↓
7. 更新memory/YYYY-MM-DD.md
   ↓
8. Git提交
   ↓
9. Git推送
   ↓
10. 记录日志
```

### 6.2 晚间回顾流程

```
1. Crontab 23:50触发
   ↓
2. 检查锁文件（避免重复执行）
   ↓
3. 读取memory/YYYY-MM-DD.md（07:00-23:50）
   ↓
4. 读取Git提交记录（07:00-23:50）
   ↓
5. 读取GitHub Issues状态
   ↓
6. 分析今日工作（已完成/进行中/待办）
   ↓
7. 总结学习知识
   ↓
8. 查漏补缺
   ↓
9. 更新memory/YYYY-MM-DD.md
   ↓
10. 更新MEMORY.md（提炼精华）
   ↓
11. 更新知识库（knowledge/YYYY-MM-DD/）
   ↓
12. Git提交
   ↓
13. Git推送
   ↓
14. 记录日志
```

## 8. 风险与应对

### 8.1 技术风险

| 风险 | 影响 | 应对措施 |
|------|------|---------|
| Crontab失效 | 高 | 添加日志监控，失败自动告警 |
| Git推送失败 | 中 | 自动重试3次，失败记录日志 |
| 磁盘空间不足 | 中 | 定期清理旧日志，保留7天 |
| 内存占用过高 | 低 | 限制处理文件数量，分批处理 |

### 8.2 业务风险

| 风险 | 影响 | 应对措施 |
|------|------|---------|
| 回顾遗漏重要内容 | 中 | 多维度检查（Git+Issue+Memory） |
| 更新覆盖重要信息 | 高 | Git版本控制，可回滚 |
| 推送冲突 | 低 | 先pull再push，自动合并 |
