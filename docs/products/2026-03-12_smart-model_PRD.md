## 1. 需求概述

### 1.1 背景
现有 smart-model-switch v1.3.0 已实现基础的消息复杂度分析和上下文监控，但缺少：
- 文件类型自动检测
- 更智能的多维度评分
- 成本优化建议
- 性能统计报表

smart-model v2.0 将在现有基础上增强这些功能。

### 1.2 目标
**核心目标**：让 AI 根据任务复杂度、文件类型、上下文使用量等因素，自动选择最优模型，提升响应质量和效率。

**具体目标**：
1. ✅ 模型切换准确率 > 90%
2. ✅ 切换响应时间 < 50ms
3. ✅ API 成本降低 30-50%
4. ✅ 用户满意度 > 4.5/5

### 1.3 范围
**包含**：
- 文件类型检测（代码/文档/图片）
- 多维度消息复杂度分析（4 维度加权）
- 上下文监控与保护（双策略系统）
- AI 主动检测机制（静默模式）
- 自动切换执行（本地 + 远程）
- 成本优化建议
- 性能统计报表

**不包含**：
- 模型训练或微调
- 自定义模型部署
- 跨平台同步（多设备）

## 2. 功能需求

### 2.1 核心功能（P0）

#### 功能 1：文件类型检测 ⭐⭐⭐
- **描述**：自动检测用户发送的文件类型
- **输入**：文件路径/内容
- **输出**：文件类型（代码/文档/图片/无文件）
- **规则**：
  - 代码文件（.py/.js/.sh/.ts/.java/.go/.rs 等）→ Coding 模型
  - 文档文件（.md/.txt/.pdf/.docx/.xlsx 等）→ 大上下文模型
  - 图片文件（.jpg/.png/.gif/.webp 等）→ Vision 模型
  - 无文件 → 根据消息复杂度判断
- **验收标准**：
  - [ ] 支持 20+ 种常见文件扩展名
  - [ ] 检测准确率 > 95%
  - [ ] 检测时间 < 10ms

#### 功能 2：多维度消息复杂度分析 ⭐⭐⭐⭐⭐
- **描述**：4 维度加权评分系统
- **输入**：用户消息文本
- **输出**：复杂度评分（0-10 分）
- **评分维度**：
  - **长度评分**（30%）：<50 字 (1 分) <200 字 (2 分) <500 字 (3 分)
  - **关键词评分**（40%）：简单 (0 分) 中等 (2 分) 复杂 (3 分)
  - **代码检测**（20%）：无代码 (0 分) 少量 (2 分) 大量 (3 分)
  - **视觉检测**（10%）：无视觉 (0 分) 有视觉 (3 分)
- **模型映射**：
  - 0-3 分 → Flash（轻量模型）
  - 4-6 分 → Main（标准模型）
  - 7-8 分 → Complex（强模型）
  - 9-10 分 → Complex + 深度分析模式
- **验收标准**：
  - [ ] 评分准确率 > 90%
  - [ ] 分析时间 < 50ms
  - [ ] 支持自定义关键词库

#### 功能 3：上下文监控与保护 ⭐⭐⭐⭐⭐
- **描述**：监控当前会话的上下文使用量
- **输入**：session_status 输出
- **输出**：上下文占用率 + 切换建议
- **规则**：
  - <60% → 当前模型继续 ✅
  - 60-85% → 准备切换，提醒用户 ⚠️
  - >85% → 强制切换到新会话 + 大上下文模型 🚨
- **防频繁切换机制**：
  - 冷却期：切换后 10 分钟内不再检测
  - 连续性要求：连续 2 次超过阈值才触发
  - 通知间隔：5 分钟内不重复提醒
- **验收标准**：
  - [ ] 监控准确率 100%
  - [ ] 冷却期执行率 100%
  - [ ] 上下文保护成功率 > 95%

#### 功能 4：AI 主动检测机制 ⭐⭐⭐⭐⭐（核心创新）
- **描述**：每次 AI 回复前自动检查
- **触发时机**：AI 回复前的钩子（hook）
- **执行逻辑**：
  1. 检查上下文使用率
  2. 分析消息复杂度
  3. 判断是否需要切换模型
  4. 输出决策（静默或提醒）
- **防骚扰机制**：
  - 静默模式：无问题时不输出
  - 冷却期：切换后 10 分钟内不检测
  - 连续性要求：连续 2 次超过阈值才触发
- **验收标准**：
  - [ ] 静默模式覆盖率 80%（无问题时不输出）
  - [ ] 检测准确率 > 90%
  - [ ] 用户无感知（不打断对话流）

#### 功能 5：自动切换执行 ⭐⭐⭐⭐
- **描述**：根据规则自动切换模型
- **输入**：模型切换决策
- **输出**：更新配置或发送系统事件
- **执行模式**：
  - **本地模式**：修改 agents.json（session-memory 集成）
  - **远程模式**：发送系统事件到 Gateway（cron 集成）
  - **混合模式**：优先本地，失败时远程
- **验收标准**：
  - [ ] 切换成功率 100%
  - [ ] 切换时间 < 1 秒
  - [ ] 配置持久化（重启后保持）

### 2.2 辅助功能（P1）

#### 功能 6：成本优化建议
- **描述**：根据 API 使用量提供成本优化建议
- **输入**：API 调用记录
- **输出**：优化建议报告（每周）
- **验收标准**：
  - [ ] 建议可行性 > 80%
  - [ ] 实际节省 > 30%

#### 功能 7：性能统计报表
- **描述**：统计各模型的响应时间、质量评分
- **输入**：历史响应记录
- **输出**：性能报表（每日/每周）
- **验收标准**：
  - [ ] 数据准确性 100%
  - [ ] 报表可读性 > 4.5/5

## 3. 技术需求

### 3.1 技术栈
- **语言**：Bash + Node.js + Python
- **核心依赖**：
  - session-memory-enhanced v4.0.0
  - context-manager v2.2.2
  - smart-memory-sync v1.0.0
  - smart-model-switch v1.3.0（参考实现）
- **平台**：Linux/macOS
- **网络**：无需外部 API（本地分析）

### 3.2 性能指标
- **响应时间**：< 50ms（分析速度）
- **切换时间**：< 1 秒（模型切换）
- **准确率**：> 90%（模型选择）
- **可用性**：> 99%

### 3.3 安全要求
- [x] 无外部 API 调用（本地执行）
- [x] 无敏感数据存储
- [x] 配置本地化（agents.json）
- [x] 日志脱敏（隐藏 API Key）
