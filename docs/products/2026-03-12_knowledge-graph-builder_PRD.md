## 1. 需求概述

### 1.1 背景
知识管理需要结构化，知识图谱可以可视化知识关系，提升知识检索效率。
基于论坛最佳实践，整合以下思路：
- 自动化知识提取（减少手动整理 90% 时间）
- 智能关系发现（AI 驱动，准确率>85%）
- 交互式可视化（支持探索式学习）
- 与现有知识库集成（QMD+MEMORY）

### 1.2 目标
- ✅ 知识自动提取（支持 Markdown/PDF/HTML）
- ✅ 关系识别（AI 驱动，准确率>85%）
- ✅ 图谱可视化（Web UI，支持交互）
- ✅ 语义搜索（自然语言查询）
- ✅ 与 OpenClaw 深度集成

## 2. 功能需求

### 2.1 核心功能（Phase 1 - MVP）

#### 1️⃣ 知识提取
- ✅ **文档解析**
  - Markdown 解析（原生支持）
  - PDF 解析（使用 PyPDF2）
  - HTML 解析（使用 BeautifulSoup）
  - 纯文本解析
  
- ✅ **实体识别**
  - 人名、地名、组织名
  - 技术术语、概念
  - 时间、日期
  - 自定义实体类型
  
- ✅ **属性提取**
  - 实体描述
  - 实体分类
  - 关键属性

#### 2️⃣ 关系识别
- ✅ **实体关系抽取**
  - 基于规则的初步抽取
  - AI 增强的关系识别
  - 关系置信度评分
  
- ✅ **关系类型分类**
  - 上下位关系（is-a）
  - 组成关系（part-of）
  - 关联关系（related-to）
  - 因果关系（causes）
  - 时间关系（before/after）
  
- ✅ **关系强度评估**
  - 强关系（置信度>0.8）
  - 中关系（置信度 0.5-0.8）
  - 弱关系（置信度<0.5）

#### 3️⃣ 图谱存储
- ✅ **图数据库**
  - Neo4j 社区版（推荐）
  - 或 SQLite + NetworkX（轻量级）
  
- ✅ **数据模型**
  - 节点（Node）：实体
  - 边（Edge）：关系
  - 属性（Property）：元数据

### 2.2 增强功能（Phase 2）

#### 4️⃣ 图谱可视化
- ✅ **Web UI**
  - 力导向图展示
  - 节点缩放/拖拽
  - 关系高亮
  
- ✅ **交互式探索**
  - 点击节点查看详情
  - 路径查找（两点间关系）
  - 邻居节点展开
  
- ✅ **导出功能**
  - 导出为 PNG/SVG
  - 导出为 JSON/GraphML
  - 导出为 Markdown 报告

#### 5️⃣ 语义搜索
- ✅ **自然语言查询**
  - "查找所有与 Python 相关的技术"
  - "显示 A 和 B 之间的关系"
  - "找出影响最大的概念"
  
- ✅ **关联推荐**
  - 相关节点推荐
  - 潜在关系发现
  - 知识缺口提示
  
- ✅ **路径查找**
  - 最短路径
  - 所有路径
  - 加权路径

### 2.3 集成功能（Phase 3）

#### 6️⃣ OpenClaw 集成
- ✅ **与 QMD 集成**
  - 自动索引知识库
  - 图谱更新同步
  
- ✅ **与 MEMORY 集成**
  - 记忆图谱化
  - 记忆关联发现
  
- ✅ **与 Git 集成**
  - 图谱版本管理
  - 变更追踪

## 9. 使用示例

### 9.1 CLI 使用

```bash
# 构建知识图谱
./skill.sh build --input docs/ --output graph.json

# 查询图谱
./skill.sh query "查找所有与 Python 相关的技术"

# 可视化
./skill.sh visualize --open

# 导出报告
./skill.sh export --format markdown --output report.md
```

### 9.2 API 使用

```python
from kg_builder import KnowledgeGraph

kg = KnowledgeGraph()

# 添加文档
kg.add_document("docs/python_intro.md")

# 查询
results = kg.search("Python 的相关技术")

# 可视化
kg.visualize(output="graph.html")
```
