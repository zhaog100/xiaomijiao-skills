## 1. 需求概述

### 1.1 背景
双米粒协作系统已经成功开发了 10+ 个技能（context-manager、smart-memory-sync、smart-model-switch 等），但缺少一个**演示技能**来展示完整的协作流程。

新成员（或官家）想了解：
- 小米辣（PM）和小米粒（Dev）如何协作？
- 从概念到发布的完整流程是什么？
- 双向 Review 机制如何运作？

demo-skill 就是为了回答这些问题而存在的"元技能"。

### 1.2 目标
**核心目标**：演示双米粒协作系统的完整流程

**具体目标**：
1. ✅ 展示从 Concept → PRD → Issue → 开发 → Review → 发布的完整流程
2. ✅ 建立可复用的开发模板
3. ✅ 验证双向协作机制的有效性
4. ✅ 为后续技能开发提供参考

### 1.3 范围
**包含**：
- GitHub Issue 创建和管理
- Git 分支管理（feature 分支 → master）
- 双向 Review 流程（12 维度评价 + 质疑清单）
- ClawHub 发布流程
- 简单的演示功能（命令行交互）

**不包含**：
- 复杂的功能实现（保持简单）
- 外部 API 集成
- 数据库操作
- 图形界面

## 2. 功能需求

### 2.1 核心功能

**功能 1：技能初始化**
- **描述**：运行技能时显示欢迎信息和可用命令
- **输入**：用户执行 `demo-skill` 命令
- **输出**：
  ```
  🌾 欢迎使用 demo-skill！
  
  这是双米粒协作系统的演示技能。
  
  可用命令：
    demo-skill status    - 显示技能状态
    demo-skill info      - 显示技能信息
    demo-skill help      - 显示帮助信息
  ```
- **验收标准**：
  - [ ] 命令可执行
  - [ ] 输出格式正确
  - [ ] 响应时间 < 1 秒

**功能 2：状态查询**
- **描述**：显示双米粒协作系统的当前状态
- **输入**：用户执行 `demo-skill status` 命令
- **输出**：
  ```
  📊 双米粒协作系统状态
  
  小米辣（PM）：在线
  小米粒（Dev）：在线
  协作模式：双向 Review
  
  已发布技能：10+
  待开发技能：10
  协作效率：⭐⭐⭐⭐⭐
  ```
- **验收标准**：
  - [ ] 状态信息准确
  - [ ] 格式美观
  - [ ] 响应时间 < 1 秒

**功能 3：技能信息展示**
- **描述**：显示技能本身的元信息
- **输入**：用户执行 `demo-skill info` 命令
- **输出**：
  ```
  📦 demo-skill 信息
  
  版本：v1.0.0
  ClawHub ID: k97xxxxx
  创建日期：2026-03-12
  开发者：小米辣 + 小米辣
  技术栈：Bash + Git + GitHub Issues
  
  协作流程：
  1. 小米辣创建 PRD
  2. 小米辣开发实现
  3. 双向 Review
  4. 发布到 ClawHub
  ```
- **验收标准**：
  - [ ] 信息完整
  - [ ] 版本号准确
  - [ ] 协作流程描述清晰

### 2.2 扩展功能

**功能 4：帮助信息**
- **描述**：显示详细的使用帮助和协作流程说明
- **输入**：用户执行 `demo-skill help` 命令
- **输出**：详细的使用说明和双米粒协作流程文档
- **验收标准**：
  - [ ] 帮助信息完整
  - [ ] 包含示例命令
  - [ ] 包含协作流程说明

## 8. 协作流程说明

### 8.1 小米辣（PM）职责
1. ✅ 创建 Concept 文档
2. ✅ 编写 PRD 文档（本文档）
3. ⏳ 创建 GitHub Issue
4. ⏳ Review 开发成果（12 维度评价）
5. ⏳ 5 层验收
6. ⏳ 批准发布

### 8.2 小米粒（Dev）职责
1. ⏳ Git 拉取最新代码
2. ⏳ 查询 Issue
3. ⏳ 技术设计
4. ⏳ 开发实现
5. ⏳ 自检清单
6. ⏳ 评论 Issue
7. ⏳ Git 提交并推送
8. ⏳ 发布到 ClawHub

### 8.3 双向 Review 机制
**小米辣 → 小米辣**：
- 12 维度评价（功能完整性、代码质量、文档完整性等）
- 反对意见（如有）
- 改进建议

**小米辣 → 小米辣**：
- 质疑清单（PRD 是否有歧义、需求是否合理）
- 补充建议（技术角度的优化）
- 风险评估
