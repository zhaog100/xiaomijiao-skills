**建立时间**：2026-03-12 12:56

---

## 📋 反馈触发条件

### 必须反馈的状态变化

1. ✅ **开发完成** → 反馈成果和Review请求
2. ✅ **Review完成** → 反馈批准/拒绝结果
3. ✅ **发布完成** → 反馈ClawHub ID和链接
4. ✅ **开发停止** → 反馈停止原因和进度
5. ✅ **遇到阻塞** → 反馈问题和求助
6. ✅ **每日回顾** → 反馈今日成果和学习

---

## 🔄 反馈方式

### 方式1：GitHub Issue评论（推荐）⭐⭐⭐⭐⭐

**位置**：当前Issue评论区

**优势**：
- ✅ 米粒儿自动检测（每5分钟）
- ✅ 历史记录完整
- ✅ 可追溯

**用法**：
```bash
gh issue comment [number] --body "反馈内容"
```

---

### 方式2：通知文件

**位置**：`/tmp/notify_mili.txt`

**优势**：
- ✅ 米粒儿自动检测
- ✅ 简单快速

**用法**：
```bash
echo "SKILL_NAME=demo-skill
STATUS=completed
NEXT_STEP=review" > /tmp/notify_mili.txt
```

## 💡 反馈原则

1. **及时性**：状态变化立即反馈
2. **完整性**：包含所有关键信息
3. **清晰性**：结构清晰，易于理解
4. **可操作性**：明确下一步行动
