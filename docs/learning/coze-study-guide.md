## 1. Coze 是什么

### 概述

**Coze（扣子）** 是字节跳动推出的 **AI Bot/Agent 开发平台**，2023年海外版（coze.com）和国内版（coze.cn）同步上线。

**核心理念**：让不会写代码的人也能创建 AI Bot，让会写代码的人能快速搭建复杂 AI Agent。

### 架构

```
用户 → [渠道] → Coze 平台 → [LLM + 插件 + 知识库 + 工作流] → 回复
                                    ↓
                              [长期记忆]
                              [数据库]
                              [变量]
```

### 核心概念

| 概念 | 说明 |
|------|------|
| **Bot** | 一个 AI 智能体，有自己的 Prompt、技能和配置 |
| **Prompt** | Bot 的系统提示词，定义角色和行为 |
| **插件（Plugin）** | 预置或自定义的工具能力（搜索、API调用等） |
| **工作流（Workflow）** | 可视化编排的多步骤任务流程 |
| **知识库（Knowledge）** | 上传文档/URL，Bot 基于 RAG 检索回答 |
| **记忆（Memory）** | 短期对话记忆 + 长期用户偏好记忆 |
| **变量（Variables）** | Bot 参数，支持开场白、Prompt 模板变量 |
| **数据库（Database）** | 结构化数据存储，Bot 可查询 |
| **多智能体（Multi-Agent）** | 多个 Bot 协作完成复杂任务 |

### 两个版本

| 维度 | coze.cn（国内版） | coze.com（国际版） |
|------|-------------------|-------------------|
| 模型 | 豆包、Doubao、智谱、DeepSeek | GPT-4o、Claude、Gemini |
| 渠道 | 微信、飞书、钉钉、企微、抖音 | Discord、Telegram、WhatsApp |
| 合规 | 国内数据合规 | 全球可用 |
| 价格 | 免费 + 按量计费 | 免费 + 按量计费 |
| 推荐场景 | **国内客户首选** | 海外客户 |

## 2. 平台核心功能

### 2.1 Bot 创建

1. **选模型** → 2. **写 Prompt** → 3. **加技能** → 4. **调参** → 5. **发布**

- **Prompt 编辑器**：支持变量插入 `{{variable}}`，支持富文本
- **模型选择**：豆包系列（轻量/标准/高级）、第三方模型
- **调试面板**：实时对话测试，查看日志和 Token 消耗

### 2.2 工作流（Workflow）

**可视化拖拽编排**，支持：

- **LLM 节点**：调用大模型
- **代码节点**：执行 Python/JavaScript
- **条件分支**：if/else 逻辑判断
- **循环节点**：批量处理
- **HTTP 请求**：调用外部 API
- **知识库检索**：RAG 查询
- **插件调用**：触发预置插件
- **变量节点**：数据转换

**典型工作流示例**：
```
用户输入 → 意图识别(LLM) → [客服问题] → 知识库检索 → 生成回复
                           → [投诉建议] → 记录到数据库 → 通知客服
                           → [下单咨询] → 调用订单API → 返回状态
```

### 2.3 插件系统

**预置插件**（100+）：
- 互联网搜索（Google、Bing）
- 图片生成（DALL-E、Midjourney）
- 文件处理（PDF、Excel、PPT）
- 地图、天气、翻译
- 飞书、钉钉、企业微信集成

**自定义插件**：
- 通过 OpenAPI Schema 定义
- 支持 OAuth2 认证
- 可发布到插件市场

### 2.4 知识库

- **数据源**：TXT、PDF、Word、Excel、PPT、Notion、网页 URL
- **分块策略**：自动分段 + 手动调整
- **检索方式**：语义检索 + 关键词检索 + 混合检索
- **更新**：支持增量更新，文件变化自动重新索引
- **单 Bot 可关联多个知识库**

### 2.5 记忆系统

- **对话记忆**：自动保存最近对话上下文
- **长期记忆**：自动提取用户偏好、重要事实
- **变量记忆**：通过 Variables 存储用户数据

### 2.6 多智能体（Multi-Agent）

- **主 Bot + 子 Bot** 架构
- 主 Bot 负责意图路由，分发到专业子 Bot
- 子 Bot 各自独立 Prompt + 知识库 + 技能

| 维度 | **Coze** | **Dify** | **FastGPT** | **GPTs** | **n8n AI** |
|------|----------|----------|-------------|----------|------------|
| **定位** | AI Bot 平台 | LLM 应用开发 | 知识库问答 | OpenAI Bot | 自动化+AI |
| **上手难度** | ⭐ 极低 | ⭐⭐ 中等 | ⭐⭐ 中等 | ⭐ 低 | ⭐⭐⭐ 较高 |
| **工作流** | ✅ 可视化 | ✅ 可视化 | ✅ 基础 | ❌ | ✅ 强大 |
| **知识库/RAG** | ✅ 优秀 | ✅ 好 | ✅ 优秀 | ✅ 基础 | ⚠️ 需插件 |

| **自定义插件** | ✅ OpenAPI | ✅ API+工具 | ⚠️ 有限 | ✅ Actions | ✅ 原生 |
| **多智能体** | ✅ 原生支持 | ⚠️ 工作流模拟 | ❌ | ❌ | ✅ 可编排 |
| **自部署** | ❌（企业版可） | ✅ 开源 | ✅ 开源 | ❌ | ✅ 自部署 |
| **代码自由度** | ⚠️ 有限 | ✅ 高 | ⚠️ 中等 | ❌ | ✅ 完全自由 |
| **中国生态** | ✅ 最佳 | ✅ 好 | ✅ 好 | ❌ | ⚠️ 需配置 |
| **免费额度** | 慷赠 | 开源免费 | 开源免费 | GPT Plus | 开源免费 |

### 结论

- **快速交付、多渠道、中国客户** → **Coze 首选**
- **需要自部署、数据敏感** → Dify/FastGPT
- **复杂自动化编排** → n8n + Coze 组合
- **Coze 做前端 Bot，n8n 做后端自动化** → 黄金组合

## 4. 可构建的 Bot 类型

### 4.1 自动客服 Bot

**适用行业**：电商、餐饮、教育、SaaS

**核心能力**：
- 基于 FAQ 知识库自动回答
- 意图识别 + 路由（售前/售后/投诉）
- 订单查询（对接ERP API）
- 人工转接机制

**Coze 实现**：
```
Prompt: 你是XX品牌的智能客服...
知识库: 产品手册、FAQ、退换货政策、价格表
插件: 订单查询API、优惠券查询API
工作流: 意图分类 → [售前]知识库回答 / [售后]调API查询 / [投诉]转人工
变量: 用户ID、订单号
渠道: 微信公众号/小程序、飞书、钉钉
```

### 4.2 知识库问答 Bot

**适用行业**：企业内部、法律、医疗、教育

**核心能力**：
- 上传企业文档（规章制度、操作手册、产品文档）
- 语义搜索 + 精准回答
- 来源溯源（标注引用段落）

**Coze 实现**：
```
知识库: 上传全部PDF/Word文档（支持100+文件）
检索模式: 混合检索（语义+关键词）
Prompt: 基于以下资料回答，如不确定请说明...
插件: 无需额外插件
渠道: 飞书机器人、企业微信、Web SDK
```

### 4.3 工作流自动化 Bot

**适用场景**：数据报表、审批流程、内容发布

**核心能力**：
- 定时触发（每天/每周）
- 数据采集 + 处理 + 输出
- 多步骤编排

**Coze 实现**：
```
工作流:
  定时触发 → 读取数据库 → LLM分析 → 生成报告 → 发送飞书/邮件
  
  示例：日报生成
  18:00 触发 → 查询销售数据库 → LLM 总结趋势 → 生成 Markdown → 推送到群
```

### 4.4 内容生成 Bot

**适用场景**：电商文案、社交媒体、营销素材

**核心能力**：
- 批量生成文案
- 多风格/多平台适配
- 图片生成（结合 DALL-E）

**Coze 实现**：
```
Prompt: 你是一位资深电商文案专家...
变量: 产品名称、卖点、目标平台、风格
工作流: 输入产品信息 → 生成小红书文案 → 生成朋友圈文案 → 生成抖音标题
插件: 图片生成（可选）
```

### 4.5 数据处理 Bot

**适用场景**：数据分析、报表生成、信息提取

**核心能力**：
- 上传 Excel/CSV → AI 分析
- 自然语言查数据
- 自动生成图表描述

**Coze 实现**：
```
工作流:
  上传Excel → 代码节点解析数据 → LLM分析 → 生成文字报告
  
数据库: 存储结构化业务数据
LLM: 用自然语言查询数据库
```

## 6. API 与集成

### Bot API（v3）

```bash
# 发送消息
curl -X POST 'https://api.coze.cn/v3/chat' \
  -H 'Authorization: Bearer YOUR_TOKEN' \
  -H 'Content-Type: application/json' \
  -d '{
    "bot_id": "BOT_ID",
    "user_id": "USER_123",
    "stream": false,
    "auto_save_history": true,
    "additional_messages": [{
      "role": "user",
      "content": "你好",
      "content_type": "text"
    }]
  }'
```

### 工作流 API

```bash
# 触发工作流
curl -X POST 'https://api.coze.cn/v1/workflow/run' \
  -H 'Authorization: Bearer YOUR_TOKEN' \
  -H 'Content-Type: application/json' \
  -d '{
    "workflow_id": "WORKFLOW_ID",
    "parameters": {
      "key": "value"
    }
  }'
```

### 与外部系统集成方式

1. **自定义插件**（推荐）→ OpenAPI Schema 导入
2. **工作流 HTTP 节点** → 直接调外部 API
3. **Webhook** → 外部系统触发 Coze 工作流
4. **Bot API** → 外部系统调用 Coze Bot

## 8. 商业应用场景

### 场景1：自动客服（最易成单）

**目标客户**：电商店铺、教育培训、本地服务

**痛点**：
- 客服人力成本高（¥5,000-8,000/人/月）
- 夜间无人值班
- 重复问题占用时间

**解决方案**：
```
Bot 功能：
✅ 7×24 自动应答
✅ FAQ 知识库（产品/价格/物流/售后）
✅ 意图识别分流
✅ 订单状态查询
✅ 优惠券发放
✅ 满意度收集
```

**效果承诺**：
- 回答 80% 的常见问题
- 响应时间 < 3秒
- 减少 50% 人工客服工作量

### 场景2：智能文档助手

**目标客户**：律所、财务公司、制造业、政府

**痛点**：
- 企业文档散落各处，新人找不到
- 制度流程更新后老版本仍在流传
- 重复解答内部政策问题

**解决方案**：
```
Bot 功能：
✅ 上传全部企业文档（制度/流程/模板）
✅ 语义搜索 + 精准回答
✅ 标注来源段落
✅ 权限控制（不同部门不同知识库）
✅ 飞书/企微直接提问
```

### 场景3：HR 招聘助手

**目标客户**：中小企业 HR

**痛点**：
- 简历筛选耗时
- 候选人反复问相同问题
- 面试安排沟通成本

**解决方案**：
```
Bot 功能：
✅ 岗位JD自动生成
✅ 候选人自动问答（薪资/福利/地点）
✅ 简历解析 + 初筛评分
✅ 面试时间自动预约
✅ 面试提醒通知
```

### 场景4：营销内容生成

**目标客户**：电商、MCN、自媒体

**痛点**：
- 内容产出效率低
- 多平台风格不同
- 文案质量不稳定

**解决方案**：
```
Bot 功能：
✅ 一键生成多平台文案（小红书/朋友圈/抖音）
✅ 基于产品库自动适配卖点
✅ SEO 关键词优化
✅ 批量生成（上传商品列表 → 批量输出）
```

### 场景5：数据分析报告

**目标客户**：运营团队、管理层

**痛点**：
- 日报/周报撰写耗时
- 数据分析需要技术背景
- 报告格式不统一

**解决方案**：
```
Bot 功能：
✅ 定时读取业务数据
✅ LLM 自动分析趋势/异常
✅ 生成结构化报告
✅ 自动推送飞书/钉钉群
```

## 10. 获客策略

### 10.1 小红书内容营销（最推荐）

**目标人设**：AI 自动化顾问 / 效率提升专家

## 11. 局限性与应对方案

### Coze 做不到的

| 局限 | 说明 | 解决方案 |
|------|------|----------|
| **不能自部署** | 数据在字节服务器 | 敏感数据用 Dify 自部署，Coze 做前端 |
| **工作流能力有限** | 复杂循环/并行受限 | 用 n8n 做复杂编排，Coze Bot 作为节点 |
| **定制化受限** | UI/UX 不能深度定制 | Web SDK + 自定义前端 |
| **模型限制** | 不能用任意模型 | 用 API 模式换自己的模型 |
| **无法离线运行** | 必须联网 | 无解，但大多数场景不需要 |

### Coze + n8n 黄金组合

```
[用户] → [Coze Bot] 智能对话层
           ↓ API调用
        [n8n 工作流] 自动化编排层
           ↓
        [企业系统] ERP/CRM/数据库
```

**分工**：
- **Coze**：自然语言理解、对话管理、知识库问答
- **n8n**：定时任务、数据同步、多系统集成、审批流程

### Coze + Dify 组合

```
[对外客户] → Coze Bot（多渠道、快速上线）
[企业内部] → Dify（私有部署、数据安全）
```
