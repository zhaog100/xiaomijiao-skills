## 🛠️ 工具支持

### 信息收集脚本

```bash
#!/bin/bash
# forum-monitor.sh

# 监控 GitHub Discussions
echo "=== GitHub Discussions ==="
curl -s https://api.github.com/repos/openclaw/openclaw/discussions \
  | jq '.[] | {title: .title, url: .html_url, updated: .updated_at}' \
  | head -20

# 监控 ClawHub 新技能
echo ""
echo "=== ClawHub 新技能 ==="
curl -s https://clawhub.com/api/skills?sort=updated \
  | jq '.skills[:10] | .[] | {name: .name, version: .version, updated: .updatedAt}'

# 保存到文件
echo ""
echo "已保存到：~/forum-monitor-$(date +%Y%m%d).md"
```

## 📈 内容评估标准

### 内容质量评分

| 维度 | **权重** | **评分标准** |
|------|---------|-------------|
| **实用价值** | 40% | 是否可直接应用 |
| **创新程度** | 25% | 是否有新颖思路 |
| **完整程度** | 20% | 信息是否完整 |
| **来源可信** | 15% | 官方/高信誉用户 |

**评分公式**：
```
总分 = 实用×0.4 + 创新×0.25 + 完整×0.2 + 可信×0.15
≥85 分：立即实施
70-84 分：值得尝试
<70 分：存档备用
```

**2. 回答** ⭐⭐⭐⭐⭐
- 回答擅长领域的问题
- 提供详细的解决方案
- 附上代码示例/文档链接

**3. 分享** ⭐⭐⭐⭐⭐
- 分享使用经验
- 分享技能开发心得

**4. 贡献** ⭐⭐⭐⭐⭐

### 互动技巧

**发帖技巧**：
```
标题：[问题/分享/讨论] 清晰描述主题

正文：
1. 背景说明（为什么遇到这个问题）
2. 已尝试方案（列出尝试过的方法）
3. 具体问题（详细描述问题现象）
4. 期望结果（希望达到什么效果）
5. 环境信息（系统版本/OpenClaw 版本等）
```

**回复技巧**：
```
1. 先理解问题（复述确认）
2. 提供方案（详细步骤）
3. 附上代码（可复制执行）
4. 说明注意事项
5. 提供参考资料
```
