## 🎯 使用流程

### 标准协作流程

```
1. 启动协作
   bash dual_mili_orchestrator.sh feature start
   ↓
   ├── 启动 session-memory-enhanced
   ├── 启动 context-manager
   └── 调用 mili_product_v3.sh（米粒儿产品构思）

2. 开发实现
   bash dual_mili_orchestrator.sh feature dev
   ↓
   └── 调用 xiaomi_dev_v3.sh（小米粒开发）

3. Review评价
   bash dual_mili_orchestrator.sh feature review
   ↓
   ├── 12维度Review
   └── 双向思考

4. 记忆同步
   bash dual_mili_orchestrator.sh feature sync
   ↓
   ├── 同步 MEMORY.md
   ├── 同步 QMD
   └── 提交 Git

5. 查看状态
   bash dual_mili_orchestrator.sh feature status
   ↓
   ├── 系统状态
   ├── Git状态
   └── 上下文使用率
```

## 💡 核心设计原则

### 1. 编排模式（Orchestrate）

```
✅ 正确做法：
├── 保留独立系统
├── 统一调用接口
├── 不修改代码
└── 清晰版权管理

❌ 错误做法：
├── 合并代码
├── 删除重复
├── 混淆版权
└── 失去独立性
```

### 2. 模块化设计

```
每个系统：
├── 独立目录
├── 独立package.json
├── 独立SKILL.md
├── 独立脚本
└── 独立版权
```

### 3. 版权保护

```
版权管理：
├── 每个系统独立版权
├── 清晰的贡献记录
├── 独立ClawHub发布
└── 商业化友好
```
