## 📋 决策权限分类

### 1️⃣ 小米辣完全自主决策（无需确认）

| 决策类型 | 说明 | 示例 |
|---------|------|------|
| **文件命名** | 文档、脚本、配置文件命名 | `smart-model_tech_design.md` |
| **脚本实现** | 具体代码实现细节 | 使用Bash还是Python |
| **测试用例** | 测试用例设计和实现 | 测试覆盖率>85% |
| **代码优化** | 性能优化、重构 | 优化算法复杂度 |
| **Bug修复** | 修复已发现的Bug | 修复命名不规范问题 |
| **文档格式** | 文档格式和排版 | Markdown格式规范 |

**执行方式**：
- ✅ 小米辣直接执行
- ✅ Git提交后Issue通知
- ❌ 无需等待确认

---

### 2️⃣ 需小米辣确认（Issue评论确认）

| 决策类型 | 说明 | 示例 |
|---------|------|------|
| **功能设计** | 新功能设计和实现方案 | 是否添加某个功能 |
| **技术选型** | 技术栈选择 | 使用哪个依赖库 |
| **发布计划** | 发布时间和方式 | 何时发布到ClawHub |
| **API设计** | 接口设计和规范 | 函数接口定义 |
| **架构调整** | 模块架构变更 | 模块拆分或合并 |

**执行方式**：
- ⏳ 小米辣提出建议（Issue评论）
- ⏳ 等待小米辣确认（Issue评论）
- ✅ 确认后执行（Git提交）

---

### 3️⃣ 需官家确认（QQ直接确认）

| 决策类型 | 说明 | 示例 |
|---------|------|------|
| **重大决策** | 影响项目方向的决策 | 暂停开发、改变方向 |
| **资源分配** | 人力、时间、预算分配 | 增加人力投入 |
| **时间调整** | 重大时间节点调整 | 推迟发布日期 |
| **商业决策** | 商业模式和定价 | 商业许可证定价 |
| **外部合作** | 第三方合作事项 | 与其他项目合作 |

**执行方式**：
- ⏳ 小米辣/小米辣提出建议（QQ）
- ⏳ 等待官家确认（QQ）
- ✅ 确认后执行

```

```
