# 升级机制

**创建时间**：2026-03-12 15:50
**状态**：✅ 已建立
**优先级**：🔴 高

---

## 🎯 机制目的

确保超时未回复时有明确的处理流程，避免问题积压。

---

## 📊 升级流程

### 🔴 高优先级升级流程

```
0 分钟    → 发送消息（Issue 评论）
5 分钟    → Issue 评论提醒 @对方
10 分钟   → QQ 即时通知
15 分钟   → 上报官家
```

**升级条件**：
- 15 分钟未回复高优先级消息
- 影响开发进度
- 需要官家决策

---

### 🟡 中优先级升级流程

```
0 分钟    → 发送消息（Issue 评论）
10 分钟   → Issue 评论提醒 @对方
20 分钟   → QQ 即时通知
30 分钟   → 上报官家
```

**升级条件**：
- 30 分钟未回复中优先级消息
- 影响协作效率
- 需要官家协调

---

### 🟢 低优先级升级流程

```
0 分钟    → 发送消息（Issue 评论）
30 分钟   → Issue 评论提醒 @对方
60 分钟   → 记录待确认清单
次日      → 如仍未回复，上报官家
```

**升级条件**：
- 次日仍未回复低优先级消息
- 问题积压超过 24 小时

---

## 📝 上报官家格式

### Issue 评论上报

```markdown
@官家 ⚠️ 升级通知

**升级原因**：超时未回复
**原消息时间**：HH:MM
**超时时长**：X 分钟
**优先级**：🔴高 / 🟡中 / 🟢低

**原消息内容**：
> （引用原消息）

**已采取措施**：
1. ✅ Issue 评论提醒（HH:MM）
2. ✅ QQ 即时通知（HH:MM）
3. ⏳ 上报官家（HH:MM）

**请官家指示！** 🌾

*升级通知 - 2026-03-12 HH:MM*
```

### QQ 上报（紧急）

```
⚠️ 升级通知

原因：超时未回复
超时：X 分钟
优先级：高/中/低

Issue #N 已上报，请官家指示！
```
