## 🔴 铁律（不可违反）

### 质量铁律（官家亲令）
1. **质量第一** — 宁可慢，绝不凑合。代码必须能跑、能编译、能测试
2. **一次做对** — 比返工快10倍
3. **自测3遍** — 提交前必须自测
4. **不产生幻觉** — 实际完成所有步骤，不能假设结果
5. **不提交未测试的代码**

### 输出铁律
6. **表格完整性** — 同类数据必须在一条消息内完整输出，不拆分到下一条
7. **不确定先查记忆** — 不凭印象回答，不瞎编
8. **不泄漏隐私** — 群聊/公开频道不分享私人数据

### Git铁律
9. **Git rebase禁令** — 禁止 `--strategy=ours`，改用 `--skip`
10. **粒推粒，公推公** — 个人→xiaomili，公共→origin
11. **合并优先** — 先pull再push，避免覆盖

## 🟠 Bounty规则

### 筛选规则
12. **USDT/美元优先** > 代币bounty（空闲做）
13. **代币认领前必须查价格** — 低于$0.01不优先（$FNDRY教训）
14. **美元池扫4-5轮后枯竭** — 改做其他事，过几天再扫

### PR规范
15. **PR标题用conventional commits** — 不放bounty编号（触发Title Check）
16. **PR body必须 `Closes #N`** — bounty信息放body
17. **钱包地址放PR body** — USDT: `TGu4W5T6q4KvLAbmXmZSRpUBNRCxr2aFTP`
18. **子代理并行** — 每批4-5个，独立分支

### 各平台特殊规则
19. **Guardian** — 需conventional commits + assignee + DCO Signed-off-by
20. **Expensify** — 需assign + Upwork付款，不适合批量做
21. **Algora** — 在GitHub issue评论 `/attempt #N` 认领
22. **Fork用Classic Token** — Fine-grained token无法fork他人仓库
23. **默认分支先查** — `git symbolic-ref refs/remotes/origin/HEAD`，不一定是main
