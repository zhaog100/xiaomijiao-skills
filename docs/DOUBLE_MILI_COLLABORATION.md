## 🎯 系统概述

双米粒协作系统是一个基于GitHub的多智能体协作框架，实现产品构思、技术实现、质量验证、发布管理的完整流程。

---

## 🎭 角色定义

### **小米辣**（官家的专业投影）

**专业领域**：
- ✅ 项目管理（PMP认证、软考高级）
- ✅ 软件测试（测试管理、测试自动化）
- ✅ 产品设计（产品设计方向）
- ✅ 客户代表（最终使用者/需求提供者）

**职责**：
1. 提供产品构思（通过GitHub）
2. 检查验证小米辣的开发成果
3. 验收通过后通知发布
4. 质量把关（测试+产品双重视角）

**核心价值**：
- 产品视角：确保功能符合用户需求
- 测试视角：确保质量达标
- 客户视角：确保用户体验良好

---

### **小米辣**（资深开发工程师）

**专业领域**：
- ✅ 软件开发（代码实现）
- ✅ 系统集成（功能整合）
- ✅ 技术实现（最佳实践）
- ✅ 发布管理（ClawHub发布）

**职责**：

**核心价值**：
- 技术实现：高质量代码

## 🔄 协作流程

```
小米辣（产品+测试+客户）        小米辣（开发+集成+发布）
        │                              │
        ├─1. 提供产品构思 ─────────────→│
        │  （通过GitHub）              │
        │                              │
        │                              ├─2. 技术实现
        │                              ├─3. 开发集成
        │                              │
        │←─4. 提交成品 ─────────────── ┤
        │  （通过GitHub）              │
        │                              │
        ├─5. 检查验证                  │
        ├─6. 测试验收                  │
        │                              │
        ├─7. 通知发布 ───────────────→ │
        │  （验收通过）                │
        │                              │
        │                              ├─8. 发布ClawHub
        │                              │
        └─✅ 完成                      └─✅ 完成
```

### **流程详解**：

1. **小米辣提供产品构思**
   - 基于专业领域设计产品
   - 创建产品构思文档（products/xxx-concept.md）
   - 提交到GitHub

2. **小米辣技术实现**
   - 读取产品构思
   - 设计技术方案
   - 编写代码实现
   - 集成功能模块

3. **小米辣提交成品**
   - 创建feature分支
   - 提交代码到GitHub
   - 通知小米辣验收

4. **小米辣检查验证**
   - 切换到对应分支
   - 检查代码质量
   - 测试功能实现
   - 验证是否符合产品构思

5. **小米辣通知发布**
   - 验收通过：创建发布通知
   - 验收不通过：提供修改建议

6. **小米辣发布上线**
   - 合并到master分支
   - 发布到ClawHub
   - 通知小米辣发布完成

## 📋 GitHub协作机制

### **仓库**：
```

```

## 📝 使用示例

### **示例1：创建测试报告生成器**

```bash
# 小米辣提供产品构思
cat > products/test-report-generator-concept.md << 'EOF'
# 产品构思：测试报告生成器

## 背景
- 测试报告编写耗时
- 需要自动化生成

## 核心功能
1. 自动收集测试数据
2. 生成标准化报告
3. 支持多种格式

## 验收标准
✅ 支持3种测试框架
✅ 生成时间<5秒
✅ 错误率<1%
EOF

git add products/
git commit -m "product: 测试报告生成器产品构思"
git push origin master

# 小米辣实现
git checkout -b feature/test-report-generator
# ... 开发实现 ...

# 小米辣验收
git checkout feature/test-report-generator
# ... 检查验证 ...

# 小米辣发布
git checkout master
git merge feature/test-report-generator
clawhub publish skills/test-report-generator
```
