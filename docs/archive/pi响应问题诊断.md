# pi/qmd响应速度问题诊断和解决方案

## 🚨 问题诊断

### 1. API密钥缺失
- **问题**: `No models available. Set API keys in environment variables.`
- **影响**: pi无法访问AI服务，导致响应缓慢或失败

### 2. 网络连接问题
- **问题**: 可能存在网络连接或防火墙限制
- **影响**: 导致pi与AI服务通信延迟

## 🔧 解决方案

### 方案1: 使用本地模型 (推荐)
```bash
# 使用本地模型避免网络依赖
pi --provider local --model local "测试消息"
```

### 方案2: 配置API密钥
```bash
# 设置环境变量
export OPENAI_API_KEY="your-api-key"
export ZAI_API_KEY="your-api-key"

# 或者设置在 ~/.pi/agent/config.json 中
```

### 方案3: 使用简化模式
```bash
# 使用只读模式，不依赖AI服务
pi --tools read,edit,write "读取知识库文档"
```

### 方案4: 优化pi配置
```bash
# 检查配置文件
cat ~/.pi/agent/settings.json

# 优化设置，减少网络请求
```
