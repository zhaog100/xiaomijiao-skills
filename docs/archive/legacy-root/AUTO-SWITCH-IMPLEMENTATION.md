# 自动会话切换实现方案

> 让技能主动切换会话的技术方案

---

## 🔍 核心问题

**用户需求**：让context-manager技能主动切换会话，无需人工干预

**技术限制**：
- /new 是用户命令，在会话内部处理
- 外部技能无法直接执行用户命令
- OpenClaw架构不允许技能"自杀"

---

## 💡 实现方案

### 方案A：通过agentTurn创建新会话（推荐）⭐

**原理**：
- 使用cron tool的agentTurn机制
- 创建一个新的isolated会话
- 新会话自动加载MEMORY.md
- 旧会话保持，新会话接管

**实现**：
```bash
# 1. 检测到85%阈值
# 2. 保存当前会话记忆（MEMORY.md + daily log）
# 3. 通过agentTurn创建新会话
# 4. 新会话自动加载记忆，继续工作
```

**优点**：
- ✅ 完全自动化，无需人工干预
- ✅ 无缝切换，新会话加载记忆
- ✅ 技术可行，使用现有机制
- ✅ 不会丢失信息

**缺点**：
- ⚠️ 不是真正的"切换"，而是"新建"
- ⚠️ 旧会话仍然存在（但不影响使用）
- ⚠️ 需要调整使用习惯

### 方案B：通过OpenClaw CLI（待验证）

**原理**：
- OpenClaw可能有创建会话的CLI命令
- 技能调用CLI创建新会话

**问题**：
- ❌ 未找到相关CLI命令
- ❌ 需要查找OpenClaw文档
- ❌ 可能不存在此功能

### 方案C：通过QQ Bot发送命令（不可行）

**原理**：
- 技能通过message tool发送"/new"
- OpenClaw识别为命令

**问题**：
- ❌ /new 只在会话内部有效
- ❌ 外部消息不会被识别为命令
- ❌ OpenClaw安全机制阻止

## 🚀 推荐实现：agentTurn方案

### 实现步骤

**1. 修改监控脚本**
```bash
# context-monitor.sh
# 检测到85%时
if [ "$USAGE" -ge 85 ]; then
    # 保存记忆
    save_memory

    # 创建新会话
    trigger_agentTurn
fi
```

**2. 保存记忆逻辑**
```bash
save_memory() {
    # 提取当前会话关键信息
    # 更新MEMORY.md
    # 更新daily log
    # 更新HEARTBEAT.md
}
```

**3. 触发agentTurn**
```bash
trigger_agentTurn() {
    # 使用cron tool添加任务
    # 任务类型：agentTurn
    # 会话类型：isolated
    # 消息：提示加载记忆继续工作
}
```

### 使用流程

```
用户对话
  ↓
上下文达到85%
  ↓
自动保存记忆
  ↓
自动创建新会话（agentTurn）
  ↓
新会话启动，加载记忆
  ↓
继续工作（无缝）
```

**用户视角**：
- 继续对话，不知不觉
- 会话ID变了，但记忆在
- 无需手动操作

## 💡 核心洞察

**"主动切换"的本质**：
- 不是"关闭旧会话，打开新会话"
- 而是"创建新会话，自动加载记忆"
- 用户无感知，体验连续

**技术实现**：
- agentTurn机制 = 新会话创建器
- MEMORY.md = 会话间记忆桥梁
- 自动监控 = 无需人工触发

**用户价值**：
- 零操作，完全自动
- 无缝体验，记忆连续
- 提高效率，避免中断
