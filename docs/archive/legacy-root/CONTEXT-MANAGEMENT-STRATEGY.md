# 上下文管理策略

## 🎯 目标

当上下文达到95%时自动开始新会话，并通过记忆系统传递关键信息，确保连续性。

---

## 📊 监控机制

### 触发条件
- **阈值**：上下文使用率 >= 95%
- **检查时机**：每次回复前
- **数据来源**：session_status

### 上下文状态
```
正常：< 80% - 继续当前会话
警告：80-94% - 开始准备总结
临界：>= 95% - 立即执行记忆传递
```

---

## 🔄 自动化流程

### 第一步：检测（自动）
```bash
session_status
↓
解析上下文使用率
↓
如果 >= 95%，触发流程
```

### 第二步：总结当前会话（自动）
```markdown

### 完成任务
- [x] 任务1
- [x] 任务2

### 进行中任务
- [ ] 任务3（进度50%）

- [ ] 待办2（中优先级）
```

### 第三步：更新记忆系统（自动）

**1. 更新MEMORY.md**（长期记忆）
```markdown
## 关键决策（新增）

## 核心教训（新增）
```

**2. 更新memory/YYYY-MM-DD.md**（每日日志）
```markdown

- 详细工作记录

- 新发现
```

**3. 更新知识库**（如有新文件）
```bash

└── 更新文档.md
```

### 第四步：通知用户（自动）

## ⚡ 技术实现

### 检测代码（伪代码）
```python
def check_context():
    status = session_status()
    usage = status.context_usage  # 如：95k/203k = 47%
    
    if usage >= 0.95:
        trigger_memory_transfer()
        notify_user()
```

### 记忆传递代码（伪代码）
```python
def trigger_memory_transfer():
    # 1. 提取当前会话内容
    summary = extract_session_summary()
    
    # 2. 更新长期记忆
    update_memory_md(summary)
    
    # 3. 更新每日日志
    update_daily_log(summary)
    
    # 4. 更新知识库
    update_knowledge_base(summary)
```
