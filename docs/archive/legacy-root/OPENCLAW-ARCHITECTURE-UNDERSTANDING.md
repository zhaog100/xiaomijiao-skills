### 系统提示词注入顺序
1. **引擎硬编码部分** - 工具、安全、CLI、技能、记忆、工作区等
2. **项目上下文文件** - AGENTS → SOUL → TOOLS → IDENTITY → USER → HEARTBEAT
3. **尾部固定部分** - 静默回复、心跳规则、运行时信息

## 🔒 沙箱隔离机制

**沙箱工作区：** Docker容器执行工具
**子代理：** 保持沙箱隔离，无提升/宿主访问
**宿主浏览器控制：** 可配置允许/阻止
**提升权限：** off/ask/full 三级控制

**重要：** 需要沙箱外的读写 → 不要启动子代理，先询问

## 😊 表情反应指导

### 最小模式（minimal）
- 仅在真正相关时反应
- 每5-10次交流最多1次反应
- 确认重要请求、表达真实情感

### 广泛模式（extensive）
- 自由反应
- 随时表达情感和个性
- 对有趣内容做反应

**QQ Bot当前：** 未启用（inline buttons不可用）

## 🧠 推理格式规则

**格式：**
```
<reasoning>
简短的内部推理
</reasoning>
<reply>
用户可见的回复
</reply>
```

**规则：**
- 所有内部推理在 `<reasoning>` 内
- 只有 `<reply>` 内的文字会展示给用户
- 外部内容会被丢弃

## 🤖 子代理精简模式

**子代理模式下保留：**
- 工具（Tooling）
- 工具调用风格
- 安全（Safety）
- OpenClaw CLI 快速参考
- 工作区（Workspace）
- 运行时（Runtime）

**子代理模式下省略：**
- 技能、记忆召回、自更新、模型别名
- 文档、用户身份、回复标签
- 消息、语音、静默回复、心跳、项目上下文

**提示词模式：**
- `"full"` → 所有章节（主代理默认）
- `"minimal"` → 仅工具+安全+CLI+工作区+运行时（子代理）
- `"none"` → 仅一行身份声明

## 💡 核心理解

### 1. 架构分层清晰
- **引擎层** - 硬编码，不可修改
- **配置层** - 文件可自定义
- **运行时层** - 动态信息

### 2. 加载顺序有逻辑
- 先加载行为规范（AGENTS）
- 再加载人格定义（SOUL）
- 最后加载工具备忘（TOOLS）

### 3. 沙箱安全重要
- 默认沙箱隔离
- 子代理保持隔离
- 需要外部访问先询问

### 4. 反应要克制
- 最小模式：5-10次1次反应
- 广泛模式：随时可反应
- 避免过度反应

### 5. 推理要内部化
- 用户看不到 `<reasoning>` 内容
- 只展示 `<reply>` 部分
- 思考过程透明可控

## 🎯 实践应用

### 会话启动
1. 读取 AGENTS.md（行为规范）
2. 读取 SOUL.md（人格定义）
3. 读取 IDENTITY.md（身份标识）
4. 读取 USER.md（用户画像）
5. 读取 TOOLS.md（工具备忘）
6. 读取 memory/YYYY-MM-DD.md（今天+昨天）
7. 如果主会话 → 读取 MEMORY.md

### 心跳执行
1. 读取 HEARTBEAT.md
2. 检查1-2项（邮件/日历/社交/天气/项目）
3. 有价值才联系
4. 无重要事项 → HEARTBEAT_OK

### 子代理启动
1. 仅加载工具+安全+CLI+工作区+运行时
2. 省略其他章节
3. 保持沙箱隔离
