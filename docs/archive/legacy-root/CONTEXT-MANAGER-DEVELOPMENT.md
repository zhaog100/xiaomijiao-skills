## 🎨 核心功能

### 1. 上下文监控
- **触发方式**：每次回复前自动检查
- **数据来源**：session_status
- **阈值**：95%

### 2. 记忆传递
**三级记忆系统：**

**紧急内容**（立即更新MEMORY.md）：
- 当前任务进度
- 未完成事项
- 重要决策
- 核心教训

**重要内容**（更新daily log）：
- 工作日志
- 技术洞察
- 问题与方案

**任务进度**（更新HEARTBEAT.md）：
- 进行中任务
- 已完成任务
- 暂缓任务

### 3. QQ消息通知
```
官家，上下文已达95%，我已完成记忆更新：

📝 本次会话总结：
- 完成：X个任务
- 进行中：Y个任务（进度Z%）
- 重要决策：N个

💾 记忆已更新：
- MEMORY.md（长期记忆）
- memory/2026-03-04.md（今日日志）
- HEARTBEAT.md（任务进度）

🔄 建议：发送 /new 开始新会话
```

## 🔧 技术实现

### 核心流程
```python
def check_context():
    """检查上下文使用率"""
    status = session_status()
    usage = status.context_usage
    
    if usage >= 0.95:
        trigger_memory_transfer()
        notify_user()

def trigger_memory_transfer():
    """触发记忆传递"""
    summary = extract_session_summary()
    update_memory_md(summary)
    update_daily_log(summary)
    update_heartbeat(summary)

def notify_user():
    """通知用户"""
    send_qq_message(message)
```

### 配置系统
```json

  "threshold": 0.95,        // 触发阈值

}
```

## 🐛 已知限制

### 限制1：无法真正自动执行
- **原因**：OpenClaw技能系统限制
- **影响**：需要我在回复时手动检查
- **解决**：已集成到工作流程

### 限制2：QQ通知依赖Gateway
- **原因**：QQ Bot需要Gateway在线
- **影响**：Gateway离线时无法通知
- **解决**：Gateway通常稳定运行

### 限制3：记忆提取依赖我的理解
- **原因**：需要AI提取关键信息
- **影响**：可能遗漏次要信息
- **解决**：优先提取紧急和重要内容
