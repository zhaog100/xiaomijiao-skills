# 智能记忆管理系统 v1.0 - 统一整合版

**版本**：v1.0

## 📋 系统概览

### 三大系统整合

| 系统 | 原定位 | 整合后角色 |
|------|--------|-----------|
| session-memory-enhanced | 记忆存储+检索 | **底层 - 记忆核心** |
| smart-memory-sync | 三库同步+预防 | **中层 - 同步协调** |
| context-manager | 上下文监控+切换 | **顶层 - 监控切换** |

```
┌─────────────────────────────────────────────┐

└─────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────┐
│  中层：同步协调（smart-memory-sync）         │
│  - 主动监控（5分钟检查）                     │
│  - 三库同步（MEMORY + QMD + Git）           │
│  - 主动切换（50%提醒 → 75%同步 → 85%切换）   │
│  - 双保险机制（预防 + 应急）                 │
└─────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────┐
│  底层：记忆核心（session-memory-enhanced）   │
│  - 结构化提取（LLM提取画像/事件/知识/决策）  │

│  - 三位一体自动化（记忆+QMD+Git）           │
│  - 多代理隔离（目录隔离 + 权限控制）        │
```

## 🎯 三层协同机制

### 1. 顶层：监控切换（context-manager）

**职责**：
- 实时监控上下文使用率
- 智能预测超限时间
- 执行切换逻辑
- 新会话加载记忆

**核心功能**：
1. **自适应监控频率**（v7.0）
   - 高活跃：2分钟
   - 中活跃：5分钟
   - 低活跃：10分钟
   - **Token节省：78%+**

2. **Token预算监控**（v1.1）
   - 每小时5000 tokens预算
   - 80%预警，100%超限
   - 工具调用优化建议

3. **意图指纹识别**（v1.0）
   - 6大意图类别
   - Warm层按需加载
   - 1小时缓存

4. **三级预警系统**
   - 70%（轻度）：提醒用户
   - 80%（重量）：建议优化
   - 90%（严重）：立即行动

5. **智能清理策略**
   - 轻度清理：<5秒
   - 中度清理：<10秒
   - 重度清理：<15秒

6. **预测性监控**
   - 活动趋势分析
   - 1-2小时提前预警
   - 会话时长监控

### 2. 中层：同步协调（smart-memory-sync）

**职责**：
- 主动监控上下文
- 三库同步
- 预防性保护
- 触发切换

**核心功能**：
1. **主动监控**
   - 每5分钟检查
   - 三级阈值触发

2. **三库同步**
   - MEMORY.md（长期记忆）
   - QMD知识库（语义检索）
   - Git库（版本保护）

3. **主动切换** ⭐
   - 50%提醒用户
   - 75%自动同步
   - 85%触发切换

4. **双保险机制**
   - Smart Memory Sync：预防性保护
   - Context Manager：应急切换

### 3. 底层：记忆核心（session-memory-enhanced）

**职责**：
- 记忆存储
- 记忆检索
- 结构化提取
- AI摘要

**核心功能**：
1. **结构化提取**
   - LLM提取用户画像
   - 事件识别
   - 知识点提取
   - 决策追踪

2. **向量检索**
   - OpenAI Embeddings
   - 语义搜索
   - 相似度计算

3. **不可变分片**
   - Token节省90%+
   - 避免重复加载

4. **三位一体自动化**
   - 记忆保存
   - QMD更新
   - Git提交

5. **多代理隔离**
   - 目录隔离
   - 权限控制

6. **AI摘要**
   - 关键词提取
   - 重要性评估

## 🔄 统一工作流程

### 正常运行（上下文 < 50%）

```
┌─────────────────┐
│ 顶层：后台监控   │ ← 10分钟检查一次（低活跃）
└─────────────────┘
         ↓
┌─────────────────┐
│ 中层：待命       │ ← 等待触发
└─────────────────┘
         ↓
┌─────────────────┐
│ 底层：正常工作   │ ← 记忆存取
└─────────────────┘
```

### 预防阶段（上下文 50-75%）

```
┌─────────────────┐
│ 顶层：中活跃监控 │ ← 5分钟检查一次
└─────────────────┘
         ↓
┌─────────────────┐
│ 中层：50%提醒   │ ← "官家，上下文50%，建议同步记忆～"
└─────────────────┘
         ↓
┌─────────────────┐
│ 底层：准备同步   │ ← 预加载记忆
└─────────────────┘
```

### 同步阶段（上下文 75-85%）

```
┌─────────────────┐
│ 顶层：高活跃监控 │ ← 2分钟检查一次
└─────────────────┘
         ↓
┌─────────────────┐
│ 中层：75%同步   │ ← 自动同步三库
│ - MEMORY.md     │
│ - QMD知识库     │
│ - Git提交       │
└─────────────────┘
         ↓
┌─────────────────┐
│ 底层：执行同步   │ ← 三位一体自动化
└─────────────────┘
```

### 切换阶段（上下文 ≥ 85%）

```
┌─────────────────┐
│ 顶层：85%触发   │ ← 严重预警
└─────────────────┘
         ↓
┌─────────────────┐
│ 中层：调用切换   │ → 通知顶层执行
└─────────────────┘
         ↓
┌─────────────────┐
│ 顶层：执行切换   │ ← 创建新会话
│ - 保存当前记忆  │
│ - 创建新会话    │
│ - 加载记忆      │
└─────────────────┘
         ↓
┌─────────────────┐
│ 新会话启动      │ ← 无缝接续
│ - 加载MEMORY.md │
│ - 加载QMD       │
│ - 恢复对话      │
└─────────────────┘
```

### 应急处理（超限错误）

```
┌─────────────────┐
│ 检测到超限错误  │ ← "model_context_window_exceeded"
└─────────────────┘
         ↓
┌─────────────────┐
│ 顶层：紧急切换   │ ← 立即行动
└─────────────────┘
         ↓
┌─────────────────┐
│ 中层：应急同步   │ ← 快速同步
└─────────────────┘
         ↓
┌─────────────────┐
│ 底层：快速保存   │ ← 核心记忆
└─────────────────┘
```

## 🛠️ 统一协作脚本

### intelligent-memory-manager.sh

**功能**：
1. 三层统一监控
2. 智能切换决策
3. 三库同步协调
4. 应急处理

**使用方法**：
```bash
# 启动智能记忆管理
bash scripts/intelligent-memory-manager.sh start

# 检查系统状态
bash scripts/intelligent-memory-manager.sh status

# 手动同步
bash scripts/intelligent-memory-manager.sh sync

# 手动切换
bash scripts/intelligent-memory-manager.sh switch

# 停止系统
bash scripts/intelligent-memory-manager.sh stop
```
