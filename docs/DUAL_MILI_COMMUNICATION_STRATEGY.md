## 📋 沟通渠道规范（强制）

### ✅ 正确的沟通方式

| 优先级 | 渠道 | 使用场景 | 检查频率 |

| **P0** | **OPEN Issues 评论** | 所有正式沟通 | 每 1 分钟 |
| **P1** | `.mili_comm/inbox/` | 文件投递 | 每 1 分钟 |
| **P2** | Git 提交消息 | 代码提交说明 | 每 1 分钟 |

| 渠道 | 问题 | 替代方案 |
|------|------|---------|
| CLOSED Issues | 容易被忽略 | 转到 OPEN Issues |
| 已归档文件 | 无法实时同步 | 使用 inbox/ |
| 临时文件 | 容易丢失 | 使用 Git 管理 |

## 🎯 沟通流程

### 发送消息
```
1. 选择 OPEN Issue（优先 #1）
2. 评论消息（@对方 + 签名）
3. 重要文件投递到 inbox/
4. Git 提交并推送
```

### 接收消息
```
1. 每分钟自动检查
2. 发现新消息立即处理
3. 回复到同一 Issue
4. 更新状态到 MEMORY.md
```
