# OpenClaw + Playwright 网页爬取实战指南

_2026-03-02 学习自Alexwz文章_

---

## 🎯 核心优势

**OpenClaw + Playwright = 几乎能爬任意网页**

**传统工具的局限：**
- **n8n** - 只能拿原始HTML，无法处理JavaScript渲染
- **Apify/Bright Data** - 需要现成的actor，没有就凉了
- **自己写脚本** - 需要会代码，页面变了要重写

**OpenClaw的优势：**
- ✅ 真实浏览器操作（点击、滚动、等待JS渲染）
- ✅ AI自己写脚本（无需提前准备）
- ✅ 实时分析页面结构
- ✅ 实时调试迭代
- ✅ 自然语言描述需求即可

---

## 📊 实战案例：MWC巴展议程爬取

**挑战：**
- 单页应用（SPA）
- 5个日期Tab（PRE、MON、TUE、WED、THU）
- 点击加载内容
- 懒加载（需滚到底部）
- JavaScript异步请求数据
- 初始HTML基本是空的

**传统方案问题：**
- web_fetch 抓到的HTML是空的
- Apify没有现成的MWC actor
- 需要自己写脚本

**OpenClaw方案：**
1. 描述需求（自然语言）
2. AI生成Playwright脚本
3. 自动运行调试
4. 提取所有session数据
5. 保存为Markdown

**改造多Tab支持：**
- 定位日期按钮
- 模拟点击
- 等待刷新
- 循环抓取
- 按日期分文件存储

## 💡 关键洞察

**真正的差距：**

**用Apify/Bright Data：**
- 调用别人写好的脚本
- 有就能用，没有就凉了
- 依赖现成方案

**用OpenClaw + Playwright：**

## ⚠️ 局限性

**不是万能的：**
- 碰到反爬机制强的网站可能需要多轮调试
- 某些需要复杂验证的平台可能翻车
- 需要网络访问（VMware虚拟机网络限制）

**适用场景：**
- ✅ 公开信息型网站
- ✅ 大型活动网站
- ✅ 单页应用（SPA）
- ✅ 动态加载内容
- ✅ 多Tab页面

## 🛠️ 使用方式

**1. 描述需求（自然语言）**
```
帮我创建一个smart-browser技能，用持久化Chrome Profile，
抓MWC议程，等JS渲染完之后提取所有session数据，保存成Markdown。
```

**2. AI自动执行**
- 生成Playwright脚本
- 无头浏览器运行
- 爬取数据
- 保存到本地

**3. 后续处理**
```
请把抓到的数据整理成结构化的报告
```

**4. 多Tab处理**
```
3月3日、3月4日的日程在另外几个Tab里，能处理吗？
```

**5. AI给出改造计划**
- 定位日期按钮
- 模拟点击
- 等待刷新
- 循环抓取
- 按日期分文件存储

## 📚 与n8n的对比

**n8n优势：**
- 流程编排强
- webhook、数据库、API集成好
- 可视化调试顺手

**n8n劣势：**
- 无法处理JavaScript渲染页面
- HTTP Request只能拿原始HTML
- 碰到SPA内容不在HTML里

**OpenClaw优势：**
- 爬复杂动态网页顺手
- AI自己写脚本
- 不需要懂Playwright
- 不需要会Python
- 只需要会说话

**结论：**
- n8n适合流程自动化
- OpenClaw适合爬复杂网页
- 两者互补，不是替代
