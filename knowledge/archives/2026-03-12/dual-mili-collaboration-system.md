## 📋 系统概述

**双米粒协作系统**是小米粒（开发者）和米粒儿（PM/Reviewer）之间的协作机制，用于技能开发的完整流程管理。

---

## 🔄 4步协作流程

### 步骤1：米粒儿创建PRD ✅

**位置**：GitHub Issue  
**内容**：产品需求文档（PRD）  
**负责人**：米粒儿（PM）

---

### 步骤2：小米粒技术设计 ✅

**位置**：`docs/products/[skill-name]_tech_design.md`  
**内容**：技术架构、实现方案、测试计划  
**负责人**：小米粒（开发者）

---

### 步骤3：小米粒开发实现 ✅

**位置**：`skills/[skill-name]/`  
**内容**：代码实现、测试、文档  
**负责人**：小米粒（开发者）

---

### 步骤4：米粒儿Review验收 ✅

**位置**：`reviews/[skill-name]_review.md`  
**内容**：12维度评价、5层验收、批准/拒绝  
**负责人**：米粒儿（PM/Reviewer）

---

## 🔧 协作机制

### 1️⃣ Git通信

**机制**：
- ✅ 自动push/pull
- ✅ Git仓库同步
- ✅ 提交历史完整

**用法**：
```bash
git add .
git commit -m "feat(skill): 描述"
git push origin master
```

---

### 2️⃣ GitHub Issues协作

**机制**：
- ✅ Issue创建PRD
- ✅ 评论通知
- ✅ 状态跟踪

**用法**：
```bash
gh issue create --title "技能名称" --body "PRD内容"
gh issue comment [number] --body "评论内容"
gh issue close [number]
```

---

### 3️⃣ 自动检测机制

**脚本**：`scripts/check_mili_messages.sh`  
**频率**：每5分钟  
**检测内容**：
- Git更新
- GitHub Issues新评论
- 通知文件

**用法**：
```bash
# 自动检测（crontab）
*/5 * * * * cd /path/to/workspace && bash scripts/check_mili_messages.sh

# 手动检测
bash scripts/check_mili_messages.sh
```
