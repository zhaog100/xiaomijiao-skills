## 二、向量搜索启用方案

### 方案1：手动下载模型（推荐）

**步骤**：

**1. 下载embedding模型**

从HuggingFace下载（需要代理）：
```
URL: https://huggingface.co/ggml-org/embeddinggemma-300M-GGUF
文件: embeddinggemma-300M-Q8_0.gguf
大小: ~300MB
```

或使用国内镜像（如果有）：
```
URL: https://hf-mirror.com/ggml-org/embeddinggemma-300M-GGUF
文件: embeddinggemma-300M-Q8_0.gguf
```

**2. 放置模型文件**

```powershell
# 创建目录
New-Item -ItemType Directory -Path "C:\Users\zhaog\.cache\qmd\models" -Force

# 复制模型文件
Copy-Item "下载路径\embeddinggemma-300M-Q8_0.gguf" "C:\Users\zhaog\.cache\qmd\models\"
```

**3. 生成向量**

```powershell
# 设置CPU强制模式
$env:QMD_FORCE_CPU="1"

# 生成向量
node "C:\Users\zhaog\.bun\install\global\node_modules\@tobilu\qmd\dist\qmd.js" embed
```

**4. 验证**

```powershell
# 检查状态
qmd status

# 应该看到：
# Vectors: 22 embedded（不再是0）
# Pending: 0 need embedding

# 测试混合搜索
qmd query "PMP认证" -c knowledge-base --hybrid
```

### 方案2：使用更小的模型

**替代模型**：

| 模型 | 大小 | 精度 | 速度 |
|------|------|------|------|
| embeddinggemma-300M-Q8_0 | 300MB | 高 | 慢 |
| embeddinggemma-300M-Q4_0 | 150MB | 中 | 中 |
| nomic-embed-text-v1 | 274MB | 高 | 中 |

**修改QMD配置**（需要修改源码）：
```javascript
// 编辑：@tobilu/qmd/dist/llm.js
const DEFAULT_EMBED_MODEL = "hf:ggml-org/embeddinggemma-300M-GGUF/embeddinggemma-300M-Q4_0.gguf";
```

## 三、性能优化

### 3.1 CPU模式优化

**当前配置**：
- CPU模式（VMware虚拟机）
- 22个文件
- 预计10-15分钟

**优化策略**：

1. **分批生成**
```powershell
# 只生成特定collection
qmd embed -c knowledge-base
```

2. **增量更新**
```powershell
# 只生成新增文件的向量
qmd embed --incremental
```

3. **闲时生成**
```powershell
# 配置cron任务在夜间生成
# 编辑：CRON-CONFIG.md
```

### 3.2 GPU加速方案

**前提条件**：
- 宿主机有NVIDIA显卡
- 配置VMware GPU直通

**步骤**：
1. 宿主机安装NVIDIA驱动
2. 配置VMware GPU直通
3. 虚拟机安装CUDA Toolkit
4. 移除 `QMD_FORCE_CPU=1`
5. 运行 `qmd embed`

**预期提升**：
- 首次生成：1-2分钟（提升50-60%）
- 增量更新：5-10秒（提升50%）

## 四、混合搜索策略

### 4.1 搜索模式对比

| 模式 | 命令 | 精度 | 速度 |
|------|------|------|------|
| 关键词 | `qmd search "query" -c kb` | 70% | 快 |
| 向量 | `qmd vsearch "query" -c kb` | 93% | 中 |
| 混合 | `qmd query "query" -c kb --hybrid` | 95% | 中 |

### 4.2 使用建议

**关键词搜索**（当前可用）：
```powershell
# 快速查询
qmd search "PMP认证" -c knowledge-base -n 3

# 适用场景：
# - 明确的关键词
# - 快速查询
# - 精确匹配
```

**向量搜索**（启用后）：
```powershell
# 语义查询
qmd vsearch "如何管理项目风险" -c knowledge-base -n 3

# 适用场景：
# - 模糊描述
# - 同义词查询
# - 概念性查询
```

**混合搜索**（推荐）：
```powershell
# 最佳效果
qmd query "项目管理和测试的结合" -c knowledge-base --hybrid -n 3

# 适用场景：
# - 复杂查询
# - 需要高精度
# - 重要决策
```

## 五、故障排查

### 5.1 模型下载失败

**症状**：
- 卡在"Gathering information"
- 超时错误
- 网络错误

**解决方案**：
1. 检查网络连接
2. 使用代理（`HTTPS_PROXY`）
3. 手动下载模型文件
4. 使用国内镜像（如果有）

### 5.2 向量生成慢

**症状**：
- 生成时间超过20分钟
- 进度卡住不动

**解决方案**：
1. 检查CPU使用率
2. 关闭其他CPU密集型任务
3. 使用更小的模型（Q4_0 vs Q8_0）
4. 分批生成

### 5.3 内存不足

**症状**：
- 内存错误
- 进程崩溃

**解决方案**：
1. 检查可用内存
2. 关闭其他应用
3. 使用更小的模型
4. 增加虚拟机内存
