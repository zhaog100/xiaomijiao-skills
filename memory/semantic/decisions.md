## 🎯 系统架构决策

### 1. Token优化策略（2026-03-01）
- **决策**：官方API优先 + AIHubMix备选
- **原因**：AIHubMix限流风险，官方稳定可靠
- **模型优先级**：
  1. zai/glm-5（智谱官方）
  2. deepseek/deepseek-chat（DeepSeek官方）
  3. AIHubMix免费模型（限流风险）

### 2. 精准检索优先级（2026-03-01）
- **决策**：QMD搜索 + 片段读取
- **效果**：节省92.5% tokens
- **流程**：
  1. 个人记忆 → memory_search()
  2. 知识库 → qmd search
  3. 其他 → 只读必要的行

### 3. Context Monitor v5.0架构（2026-03-07）
- **决策**：智能分层 + 预测性提醒
- **核心理念**：
  - 智能触发（节省78%+ token）
  - 分层监控（轻量→详细）
  - 预测性提醒（趋势分析）
  - 用户友好（建议式）
- **理论依据**：auroras_happycapy的上下文管理理论
- **阈值**：
  - 轻量：5分钟10次工具调用
  - 详细：1小时30次提醒，50次预警
  - 性能退化：50k token阈值

### 4. Memory系统架构（2026-03-07）
- **决策**：Episodic/Semantic分离
- **理论依据**：auroras_happycapy的Memory System Architecture
- **结构**：
  ```
  memory/
  ├── episodic/（原始经历）
  │   └── YYYY-MM-DD.md
  ├── semantic/（提炼知识）
  │   ├── decisions.md
  │   ├── lessons.md
  │   └── knowledge.md
  └── MEMORY-LITE.md（启动专用，2-3KB）
  ```
- **Consolidation**：每周自动Episodic → Semantic
- **时间衰减**：3个月后删除低价值记忆

## 🛠️ 技术决策

### 1. CPU强制模式（2026-02-27）
- **决策**：QMD_FORCE_CPU=1
- **原因**：VMware虚拟GPU不支持CUDA/Vulkan
- **效果**：功能完全，速度可接受

### 2. 知识库路径（2026-02-27）
- **决策**：`knowledge/` 目录
- **Collection**：knowledge-base
- **检索**：关键词搜索 + 向量搜索（双模式）

### 3. 工具调用原则（2026-03-01）
