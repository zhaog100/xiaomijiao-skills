## 📚 今日学到

### 结构化整理经验
1. **索引一致性**: 所有索引文件需同步更新，避免信息不一致
2. **路径统一**: 工作区路径统一为 `~/.openclaw-xiaomijiao/workspace/`
3. **Remote 命名**: `origin` (技能) + `xiaomijiao` (个人)，避免与小米辣混淆
4. **推送规则**: 技能→origin，个人数据→xiaomijiao，不混推
5. **版本管理**: 索引文件标注版本号，便于追踪变更

### 知识库维护经验
6. **文件统计准确性**: 每次新增/删除文件后，所有索引文件需同步更新统计数字
7. **Git 提交粒度**: 相关文件的改动打包成一个 commit，便于追溯
8. **推送前检查**: `git status` → `git add` → `git commit` → `git push` 流程不能跳
9. **记忆文件时效**: 当日记忆文件需实时更新，不要等到晚上才补
