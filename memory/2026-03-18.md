## 🔧 技术要点总结

### GitHub API 高效用法
- Token格式：`x-access-token:ghp_XXX@github.com`（避免密码认证错误）
- Fork用Classic Token（Fine-grained无法fork他人仓库）
- PR标题格式：`[BOUNTY #N] 描述`
- PR body：`Closes #N` 自动关联issue

### 子代理并行策略
- 每批4-5个子代理同时开发
- 独立分支，互不冲突
- 完成后统一创建分支→提交→推送→创建PR
- 耗时：3-5分钟/子代理，整体10-15分钟一批

### 项目发现方法
1. GitHub搜索：`label:"💎 Bounty" is:issue is:open no:assignee`
2. Algora平台：console.algora.io（SPA，需API）
3. 关键词组合：bounty + $ + help wanted
