### 双仓库推送错误记录 ⭐⭐⭐⭐⭐

**问题**：习惯性双推导致个人笔记 3 次泄露到 origin 公共仓库

**时间线**：
| 时间 | 错误 | 泄露文件 | 修复 |
|------|------|----------|------|
| 15:24 | 习惯性双推 | 7 个个人笔记 | 从 origin 删除 |
| 16:10 | archive 又双推 | 13 个文件 | 第三次删除 |
| 16:19 | 恢复提交推错 | 个人文件回 origin | 再次删除 |

**根因**：
1. 没有提交前分类，习惯性 `git add .`
2. 习惯性 `git push origin && git push xiaomila`
3. 没有真正理解"禁止同一个 commit 推两个仓库"

**正确做法**：
```bash
# ✅ 公共内容
git add STRUCTURE-INDEX.md data/bounty-*.txt
git commit -m "chore: 公共数据更新"
git push origin master

# ✅ 个人内容
git add memory/2026-*.md docs/*-assessment.md
git commit -m "docs: 今日笔记"
git push xiaomila master

# ❌ 禁止
git add .
git commit -m "更新"
git push origin master
git push xiaomila master  # 绝对禁止！
```

**固化到 MEMORY.md**：
- 推送规则详情（origin vs xiaomila）
- 三次教训记录
- 禁止同一个 commit 推两个仓库

**最终状态**：
- ✅ origin 无个人笔记
- ✅ xiaomila 个人笔记完整
- ✅ 本地文件安全
