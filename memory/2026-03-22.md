## 🔑 今日教训

### 1. GitHub API直接操作绕过clone超时
- **问题**: VPS到GitHub网络不稳定，git clone反复超时
- **方案**: 用 `gh api` 创建 blob→tree→commit→ref，完全通过API提交
- **流程**:
  ```bash
  # 获取文件SHA
  gh api repos/OWNER/REPO/contents/PATH --jq '.sha'
  # 创建blob
  CONTENT=$(base64 -w 0 file)
  BLOB=$(gh api repos/OWNER/REPO/git/blobs -f "content=$CONTENT" -f encoding=base64 --jq '.sha')
  # 创建tree（用python3生成JSON避免bash转义问题）
  python3 -c "import json; json.dump({'base_tree':SHA,'tree':[...]},open('/tmp/tree.json','w'))"
  TREE=$(gh api repos/OWNER/REPO/git/trees --input /tmp/tree.json --jq '.sha')
  # 创建commit
  COMMIT=$(gh api repos/OWNER/REPO/git/commits -f message="..." -f tree=$TREE -f "parents[]=$SHA" --jq '.sha')
  # 更新branch
  gh api -X PATCH repos/OWNER/REPO/git/refs/heads/BRANCH -f sha=$COMMIT
  ```

### 2. JSON Schema oneOf对array有歧义
- **问题**: `oneOf: [{type: array}, {description: "any"}]` 导致node link `[id, 0]` 匹配两个分支
- **方案**: 改为 `additionalProperties: true`，不限制inputs值类型

### 3. ComfyUI代码架构理解

- `main.py`: 启动流程——prestartup→custom_nodes→server→run
- 默认分支是 `master` 不是 `main`
