## 🎯 核心修复：CPU强制模式

### 问题
- VMware虚拟GPU被 `node-llama-cpp` 误判为CUDA
- 尝试编译CUDA binaries → CMake失败

### 解决方案
**修改 `@tobilu/qmd/dist/llm.js`**，新增环境变量：
```javascript
const forceCPU = process.env.QMD_FORCE_CPU === "1";

if (forceCPU) {
    llama = await getLlama({ gpu: false, logLevel: LlamaLogLevel.error });
    process.stderr.write("QMD: Force CPU mode enabled (QMD_FORCE_CPU=1)\n");
}
```

### 使用方法
```powershell
# 永久启用（已添加到PowerShell Profile）
Set-Item -Path Env:QMD_FORCE_CPU -Value "1"

# 验证
$env:QMD_FORCE_CPU  # 应输出 "1"
```

## 📚 学习记录：提示词工程

### 学习内容
**OpenClaw提示词架构设计**（十一至十五章节）

### 核心收获

**1. 沙箱隔离设计**
- Docker容器隔离工具执行
- 权限三级分级：off/ask/full
- 原则：默认沙箱，提升权限需用户批准

**2. 反应指导模式**
- 最小模式（minimal）：正式场合，5-10次交流1次反应
- 广泛模式（extensive）：轻松场合，随时反应
- 核心原则：真实表达，适度反应

**3. 推理格式分离**
```xml
<reasoning> 内部分析（用户不可见）</reasoning>
<output> 最终回复（用户可见）</output>
```
**优势**：思考透明，输出简洁

**4. 子代理精简模式**
- 保留：工具+安全+CLI+工作区+运行时
- 省略：技能+记忆+自更新+文档+身份
- **Token节省：75%**（8000 → 2000 tokens）

**5. 提示词分层架构**
```
引擎层（硬编码）→ 项目层（文件注入）→ 运行时层（动态生成）
```

### 设计哲学

**安全第一**
> 宁可多问一次，不可多越一步

**简洁至上**
> 能用5个字说清楚，绝不用10个字

**自然交互**
> 像真人一样交流，而不是机器人

### 实践应用

**已应用到当前系统**：
- ✅ 沙箱模式（默认安全）
- ✅ 反应指导（根据频道配置）
- ✅ 推理分离（内部思考vs外部输出）
- ✅ 精简模式（子代理token优化）
- ✅ 文件分层（AGENTS.md/SOUL.md/USER.md等）

**Token优化效果**：
- 子代理提示词：节省75%
- 推理分离：用户不可见推理过程
- 按需加载：技能/记忆按需读取
