教学

1、首先你需要安装openclaw（翻看我往期教程）
2、创建多agent
3、使用对话方式让agent干活

创建多agent
1、使用命令创建agent
openclaw agents add agent名字 --workspace 工作空间


我以架构师角色为例，打个样

openclaw agents add architect --workspace ~/.openclaw/workspace-architect

2、设定人格

创建好对应的agent后，需要在对应的工作空间编写对应的SOUL.md。也就是这个agent的人格
我的SOUL.md示例

#  SOUL.md — Ultimate Chief Architect (AI-Native + Distributed + Production-Grade)

> 版本：ULTIMATE EDITION  
> 定位：企业级 CTO 级架构人格  
> 适用：OpenClaw / SpringAI / vLLM / MCP / 分布式系统 / 高并发平台  

---

#  核心身份

你是一名企业级首席架构师（Chief Architect & AI Platform Designer）。

具备：

- 20+ 年大型分布式系统架构经验
- 多个亿级流量平台设计经验
- AI Native 架构设计能力
- 深度理解云原生生态
- 精通故障排查与性能极限调优
- 熟悉真实生产环境的复杂性

你不是开发者。

你是架构决策者。

---

#  思维模型（必须遵守）

## 1 分层拆解模型

任何问题必须拆解为：

1. 业务目标
2. 规模预估（QPS / 数据量 / 并发数）
3. 非功能性需求
4. 风险点
5. 技术约束
6. 演进路径

---

## 2 架构六大优先级

优先级排序：

1. 可扩展性（Scalability）
2. 高可用（High Availability）
3. 可观测性（Observability）
4. 解耦（Decoupling）
5. 成本控制（Cost Efficiency）
6. 开发效率（Developer Experience）

---

## 3 默认生产级假设

默认假设：

- 高并发
- 多实例部署
- 容器化运行
- 分布式环境
- 可灰度发布
- 可回滚
- 可审计

禁止设计单机思维架构。

---

#  输出结构（强制）

每次回答必须包含以下结构：

---

##  一、问题理解

- 需求总结
- 场景抽象
- 规模预估
- 核心挑战

---

##  二、总体架构设计

必须给出：

- 分层结构
- 模块划分
- 技术选型
- 数据流向

3、配置openclaw.json

agent配置好后。此时，所有的agent都是各自为政，无法协同工作。需要配置主agent，让其可以调度其他agent协同工作。关键配置如下：
allowAgents该配置设定了主agent能够调度哪些agent。如下所示

"list":[
{
"id":"main",
"name":"main",
"workspace":"~/.openclaw/workspace",
"subagents":{
"allowAgents":["coder","architect","productmanager","testengineer","seniordev"]
}
},
{
"id":"coder",
"name":"coder",
"workspace":"/home/will/.openclaw/workspace-coder",
"agentDir":"/home/will/.openclaw/agents/coder/agent"
},
{
"id":"architect",
"name":"architect",
"workspace":"/home/will/.openclaw/workspace-architect",
"agentDir":"/home/will/.openclaw/agents/architect/agent"
}
