## 核心组件

### 1. `scripts/check_github_issues.sh`

**功能**：
- ✅ 检查最近更新的Issues（自上次检查时间）
- ✅ 识别新评论（小米辣的回复）
- ✅ 写入通知日志（`logs/new_comments.log`）
- ✅ 轻量级，快速执行（<1秒）

**用法**：
```bash
# 手动检查
bash scripts/check_github_issues.sh

# 心跳系统自动调用（HEARTBEAT.md）
- [ ] **GitHub Issues**：检查小米辣的最新回复（30秒检查机制）⭐⭐⭐⭐⭐
```

---

### 2. HEARTBEAT.md 集成

**定期检查项**：
```markdown
## 定期检查（每次轮换 1-2 项）

- [ ] **GitHub Issues**：检查小米辣的最新回复（30秒检查机制）⭐⭐⭐⭐⭐
- [ ] 邮件、日历...
```

**心跳逻辑**：
1. 收到心跳轮询
2. 执行 `check_github_issues.sh`
3. 检查 `logs/new_comments.log`
4. 发现新评论 → 立即通知官家 + 回复

---

### 3. core/github_monitor.py（保留但不启用）

**状态**：⏸️ 暂时禁用（独立进程模式）

**原因**：
- ❌ 独立进程不稳定
- ❌ 与主会话隔离
- ✅ 保留代码，未来可能用于其他场景

## 检查频率

**心跳频率**：取决于OpenClaw配置（通常30秒-5分钟）

**建议**：
- 高频期（协作测试）：30秒检查
- 常规期：5分钟检查
- 低频期（深夜）：10分钟检查
