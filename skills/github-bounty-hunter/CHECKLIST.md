# GitHub Bounty Hunter - 操作检查清单

## 📋 修改前检查

- [ ] 分析文件依赖关系（哪些文件引用此文件）
- [ ] 备份原文件（git 或 cp）
- [ ] 确认修改范围（最小改动原则）
- [ ] 准备回滚方案

## 📋 修改后验证

- [ ] 语法检查（bash -n / python3 -m py_compile）
- [ ] 功能测试（运行一次完整流程）
- [ ] 日志检查（无 error/critical）
- [ ] 提交信息（清晰描述变更）

## 📋 提交前检查

- [ ] git status 确认变更
- [ ] git diff 审查代码
- [ ] 运行一次完整测试
- [ ] 更新版本号（如有必要）

## 📋 禁止操作

- ❌ 不备份直接删除文件
- ❌ 不测试直接提交
- ❌ 不分析依赖直接修改
- ❌ 跳过检查清单

## 📋 错误记录

每次错误记录到 ERROR_LOG.md，包括：
- 错误描述
- 根本原因
- 修复方案
- 预防措施
