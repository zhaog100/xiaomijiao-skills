# Context Manager v2.2

> 真实API监控 + 启动优化 + 无感会话切换 - 生产就绪版

## 🎯 核心特性

### ⭐ v2.2.2 新特性（生产就绪）⭐⭐⭐⭐⭐
- **真实API监控**：调用OpenClaw API获取准确上下文（解决假监控问题）
- **Cron环境修复**：添加完整PATH/HOME变量（v2.2.1）
- **快速响应**：60%阈值+5分钟间隔（v2.2.2优化，更快预警）
- **准确计算**：totalTokens / contextTokens = 真实使用率
- **飞书通知**：60%阈值自动通知，1小时冷却期

### ✅ v2.1 特性
- **启动优化**：分层读取（核心<5KB + 摘要<10KB + QMD检索）
- **MEMORY-LITE**：精简版记忆（2.5KB），启动专用
- **启动检测**：session_status自动检查，>30%预警
- **双阈值**：启动30% + 对话60%（v2.2.2优化）

### ✅ v2.0 特性
- **无感自动切换**：上下文超过60%自动创建新会话（v2.2.2优化）
- **零用户干预**：完全自动化，无需/new
- **无缝体验**：新会话自动加载记忆，对话连续

## 🚀 快速开始

### 安装
```bash
# 从ClawHub安装
clawhub install miliger-context-manager

# 或手动安装
cd ~/.openclaw/skills
tar -xzf context-manager-v2.0.0.tar.gz
cd context-manager-v2
bash install.sh
```

### 使用
**超简单 - 只需正常聊天**

1. 继续对话（监控在后台运行）
2. 达到60%阈值（自动保存记忆）
3. 创建新会话（自动）
4. 新会话加载记忆（继续工作）

**用户视角**：对话从未中断

## 📊 工作原理

```
对话进行中
  ↓
后台监控（每5分钟）
  ↓
上下文达到60%
  ↓
自动保存记忆
  ↓
触发agentTurn
  ↓
创建新会话
  ↓
加载记忆继续
```

## 💡 为什么选择v2.1

### vs v1.0
- v1.0：提醒用户手动/new

- 手动：可能忘记，有中断感
