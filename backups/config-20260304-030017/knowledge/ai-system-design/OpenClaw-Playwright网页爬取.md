# OpenClaw + Playwright 网页爬取能力

_2026-02-27 学习记录_

---

## 📊 核心要点

### OpenClaw Playwright优势
1. **真实浏览器操作** - 点击、滚动、等待JS渲染
2. **AI自动生成脚本** - 无需提前准备，实时分析页面结构
3. **处理复杂SPA** - 单页应用、多Tab、懒加载都能搞定
4. **持久化Chrome Profile** - 复用登录状态

### 对比其他工具

| 工具 | 局限性 |
|------|--------|
| **n8n** | 无法处理JS渲染页面，需接入外部服务 |
| **Apify** | 需找现成actor，没有就自己写，还要花钱 |
| **Bright Data** | 同Apify，按量计费成本高 |
| **OpenClaw** | ✅ AI实时写脚本，无"有没有现成方案"问题 |

### 典型应用场景
- 多Tab网页（会议议程、展会日程）
- 懒加载内容（需滚动到底部）
- JavaScript异步加载（SPA单页应用）
- 需登录的页面（持久化Profile）

## 🎯 实际案例

### MWC巴展议程爬取
**挑战**：
- 5个日期Tab（PRE、MON、TUE、WED、THU）
- 每个Tab有懒加载
- 数据通过JS异步请求

**OpenClaw方案**：
```
1. 创建smart-browser技能
2. 使用持久化Chrome Profile
3. 定位日期按钮 → 模拟点击 → 等待刷新
4. 循环抓取 → 按日期分文件存储
```

**结果**：
- ✅ 无需外部服务
- ✅ 全程自动化
- ✅ 数据结构化输出

## 💡 关键洞察

### OpenClaw vs 传统方案

**传统方案**：
- 需要懂Playwright/Puppeteer
- 需要写代码、维护脚本
- 页面结构变化需重新调试

**OpenClaw方案**：
- 自然语言描述需求
- AI自己生成、调试、迭代代码
- 无需懂编程，会说话就行

### 适用场景
- ✅ 公开信息型网站
- ✅ 会议/展会日程
- ✅ 多Tab懒加载页面
- ✅ SPA单页应用

### 不适用场景
- ⚠️ 反爬机制强的网站
- ⚠️ 需复杂验证的平台
- ⚠️ 需要多轮调试的场景

## 🔧 技术实现

### Playwright Skill核心能力
1. **真实浏览器** - 完整Chrome环境
2. **DOM操作** - 点击、滚动、输入
3. **等待机制** - 等待JS渲染完成
4. **数据提取** - 从渲染后的DOM提取

### 持久化Profile优势
- ✅ 复用登录状态
- ✅ 保持cookies
- ✅ 避免重复验证

## 💭 个人评价

### 优势
- ✅ AI实时生成脚本，灵活性高
