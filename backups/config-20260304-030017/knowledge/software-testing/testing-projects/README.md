## 📝 如何使用

### 1. 创建新项目

**步骤**：
1. 复制 `PROJECT-TEMPLATE.md` 到新项目目录
2. 重命名为 `README.md`
3. 填写项目信息
4. 创建子目录（test-cases、test-reports等）
5. 更新 `PROJECT-INDEX.md`

**示例**：
```bash
# 创建新项目
mkdir -p testing-projects/my-project/{test-cases,test-reports,test-plans,test-scripts,assets}

# 复制模板
cp PROJECT-TEMPLATE.md my-project/README.md

# 编辑项目信息
vim my-project/README.md
```

### 2. 管理测试用例

**测试用例文件命名规范**：
- 格式：`[模块名]-测试用例.md`
- 示例：`统计看板-测试用例.md`、`登录模块-测试用例.md`

**测试用例内容结构**：
```markdown
# [模块名] - 测试用例

**页面路径**: #/module/path
**测试日期**: YYYY-MM-DD
**编写人**: 姓名
**版本**: vX.Y

## 🧪 测试用例

### TC001 - 用例名称
- **前置条件**: ...
- **测试步骤**: ...
- **预期结果**: ...
- **优先级**: P0/P1/P2
- **自动化**: ✅/⏸️
```

### 3. 归档测试报告

**测试报告文件命名规范**：
- 格式：`YYYY-MM-DD-[项目名]-执行报告.md`
- 示例：`2026-03-03-旅行客平台-执行报告.md`

**测试报告内容结构**：
```markdown
# [项目名] - 测试执行报告

**执行日期**: YYYY-MM-DD
**执行人**: 姓名
**测试环境**: 测试/预发/生产

## 📊 测试统计
- 总用例数: X
- 通过: X
- 失败: X
- 阻塞: X
- 通过率: X%

## 📋 详细结果
...

## 🐛 缺陷列表
...

## 💡 建议
...
```

### 4. 维护项目索引

**更新 PROJECT-INDEX.md**：
每创建一个新项目，都需要在 `PROJECT-INDEX.md` 中添加索引信息。
