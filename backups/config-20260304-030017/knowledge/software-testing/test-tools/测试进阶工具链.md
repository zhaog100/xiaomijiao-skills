### 1. Pytest - Python测试组织框架

#### 核心优势
- ✅ 简洁的测试语法
- ✅ 强大的fixture机制
- ✅ 丰富的插件生态
- ✅ 参数化测试支持
- ✅ 详细的失败信息

#### 基础使用
```python
# test_example.py
import pytest

# 简单测试
def test_addition():
    assert 1 + 1 == 2

# 参数化测试
@pytest.mark.parametrize("input, expected", [
    (1, 2),
    (2, 4),
    (3, 6),
])
def test_multiply_by_two(input, expected):
    assert input * 2 == expected

# fixture使用
@pytest.fixture
def database():
    db = Database()
    db.connect()
    yield db
    db.disconnect()

def test_database_query(database):
    result = database.query("SELECT * FROM users")
    assert len(result) > 0
```

#### 高级特性
```python
# conftest.py - 共享fixture
import pytest

@pytest.fixture(scope="session")
def browser():
    driver = webdriver.Chrome()
    yield driver
    driver.quit()

# 标记测试
@pytest.mark.slow
def test_slow_operation():
    time.sleep(5)
    assert True

# 跳过测试
@pytest.mark.skip(reason="功能未实现")
def test_feature():
    pass
```

#### 常用插件
- `pytest-html`: HTML测试报告
- `pytest-cov`: 代码覆盖率
- `pytest-xdist`: 并行测试
- `pytest-mock`: Mock对象
- `pytest-django`: Django测试支持

#### 基础使用
```python
# 安装
# pip install evaluate

import evaluate

# 加载预定义指标
accuracy = evaluate.load("accuracy")
precision = evaluate.load("precision")
recall = evaluate.load("recall")
f1 = evaluate.load("f1")

# 计算指标
predictions = [0, 1, 0, 1, 1]
references = [0, 1, 0, 1, 0]

results = accuracy.compute(predictions=predictions, references=references)
print(f"准确率: {results['accuracy']}")
```

#### NLP模型评估
```python
import evaluate

# 文本分类评估
def evaluate_text_classification(predictions, references):
    metrics = {
        "accuracy": evaluate.load("accuracy"),
        "f1": evaluate.load("f1"),
        "precision": evaluate.load("precision"),
        "recall": evaluate.load("recall")
    }
    
    results = {}
    for name, metric in metrics.items():
        results[name] = metric.compute(
            predictions=predictions,
            references=references,
            average="weighted"
        )[name]
    
    return results

# 问答系统评估
def evaluate_qa(predictions, references):
    squad_metric = evaluate.load("squad")
    results = squad_metric.compute(
        predictions=predictions,
        references=references
    )
    return results

# 机器翻译评估
def evaluate_translation(predictions, references):
    bleu = evaluate.load("bleu")
    results = bleu.compute(
        predictions=predictions,
        references=references
    )
    return results
```

#### 自定义评估指标
```python
import evaluate

# 定义自定义指标
class CustomMetric(evaluate.Metric):
    def _info(self):
        return evaluate.MetricInfo(
            description="自定义评估指标",
            citation="...",
            inputs_description="...",
            features=datasets.Features({
                "predictions": datasets.Value("float"),
                "references": datasets.Value("float"),
            }),
        )
    
    def _compute(self, predictions, references):
        # 实现计算逻辑
        score = sum(abs(p - r) for p, r in zip(predictions, references)) / len(predictions)
        return {"custom_score": score}

# 使用自定义指标
custom = CustomMetric()
results = custom.compute(predictions=[1.0, 2.0], references=[1.1, 2.1])
```

```groovy
// Jenkinsfile
pipeline {
    agent any
    
    environment {
        DOCKER_IMAGE = 'test-suite:latest'
        REPORT_DIR = 'test-reports'
    }
    
    stages {
        stage('准备环境') {
            steps {
                sh 'docker build -t ${DOCKER_IMAGE} .'
            }
        }
        
        stage('运行单元测试') {
            steps {
                sh '''
                    docker run --rm \
                        -v ${WORKSPACE}/${REPORT_DIR}:/app/reports \
                        ${DOCKER_IMAGE} \
                        pytest tests/unit -v --html=reports/unit.html
                '''
            }
            post {
                always {
                    publishHTML([
                        allowMissing: false,
                        alwaysLinkToLastBuild: true,
                        keepAll: true,
                        reportDir: "${REPORT_DIR}",
                        reportFiles: 'unit.html',
                        reportName: 'Unit Test Report'
                    ])
                }
            }
        }
        
        stage('运行集成测试') {
            steps {
                sh '''
                    docker-compose -f docker-compose.test.yml up --abort-on-container-exit
                '''
            }
        }
        
        stage('运行E2E测试') {
            parallel {
                stage('Chrome测试') {
                    steps {
                        sh 'pytest tests/e2e -v --browser=chrome'
                    }
                }
                stage('Firefox测试') {
                    steps {
                        sh 'pytest tests/e2e -v --browser=firefox'
                    }
                }
            }
        }
        
        stage('安全测试') {
            steps {
                sh '''
                    # 运行OWASP ZAP扫描
                    docker run --rm -t owasp/zap2docker-stable \
                        zap-baseline.py -t https://staging.example.com
                '''
            }
        }
    }
    
    post {
        always {
            // 清理Docker资源
            sh 'docker system prune -f'
            
            // 发送测试报告
            emailext(
                subject: "测试报告 - ${env.JOB_NAME} #${env.BUILD_NUMBER}",

                    <h2>测试执行完成</h2>
                    <p>构建号: ${env.BUILD_NUMBER}</p>
                    <p>状态: ${currentBuild.currentResult}</p>
                    <p><a href="${env.BUILD_URL}testReport">查看详细报告</a></p>
                """,
                to: 'team@example.com',
                mimeType: 'text/html'
            )
        }
    }
}
```

```python
# test_monitor.py
import time
from prometheus_client import Counter, Histogram, start_http_server

# 测试开始
def test_started(test_name):
    # 记录测试开始

# 测试结束
def test_completed(test_name, status, duration):
    test_total.labels(status=status, type=test_name).inc()
    test_duration.labels(type=test_name).observe(duration)
    
    # 如果失败，发送告警
    if status == "failed":
        send_alert(test_name, duration)

# 发送告警
def send_alert(test_name, duration):
    import requests
    webhook_url = "https://hooks.slack.com/services/YOUR/WEBHOOK"
    
    payload = {
        "text": f"测试失败告警",
        "attachments": [

                "color": "danger",

                    {"title": "执行时间", "value": f"{duration:.2f}s", "short": True}
                ]
            }
        ]
    }
    
    requests.post(webhook_url, json=payload)

# 在pytest中使用
import pytest

@pytest.hookimpl(hookwrapper=True)

    report = outcome.get_result()
    
    if report.when == "call":
        status = "passed" if report.passed else "failed"
        duration = report.duration
        test_completed(item.name, status, duration)
```

## 📊 测试工具选型指南

### 按测试类型选择

| 测试类型 | 推荐工具 | 适用场景 |
|---------|---------|---------|
| 单元测试 | Pytest, Jest | 快速反馈、TDD开发 |
| UI自动化 | Playwright, Selenium | Web应用端到端测试 |
| API测试 | Postman, RestAssured | 接口集成测试 |
| AI模型测试 | Evaluate, DeepChecks | 机器学习模型验证 |
| 安全测试 | Burp Suite, OWASP ZAP | Web应用安全扫描 |
| 性能测试 | JMeter, k6 | 负载和压力测试 |

### 按团队规模选择

#### 小型团队（1-5人）
- **核心**: Pytest + Playwright
- **补充**: Postman（API测试）
- **监控**: 简单的Grafana面板
- **CI**: GitLab CI / GitHub Actions

#### 中型团队（5-20人）
- **核心**: Pytest + Playwright + Hugging Face Evaluate
- **专项**: Evidently AI（数据监控）、Burp Suite（安全）
- **工程化**: Docker + Jenkins
- **监控**: 完整的Grafana监控体系

#### 大型团队（20+人）
- **全栈**: 所有工具组合
- **定制**: 自研测试平台
- **集成**: 完整的DevOps工具链
- **监控**: 全链路可观测性
