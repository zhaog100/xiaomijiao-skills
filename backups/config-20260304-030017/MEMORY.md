```
用户语音 → Whisper识别 → AI处理 → TTS生成 → SILK编码 → QQ发送
```

**性能优化三要素**
1. Token节省：QMD精准检索 + 精简提示词

3. 响应速度：简单任务<1秒，复杂任务<3秒

**自动化测试体系**
```
测试金字塔
├── 单元测试（最大量）

测试分层
├── Layer 1: POM页面对象（封装元素和操作）

└── Layer 3: 测试套件（组织和报告）

├── 功能测试（80%）
├── 性能测试（10%）
└── 视觉回归测试（10%）
```

**Playwright核心优势**
- 多浏览器支持：Chromium、Firefox、WebKit
- 自动等待机制：智能等待元素可操作
- 网络拦截：Mock API、修改请求响应

- 截图和视频：测试失败自动截图
- 并行执行：多测试并行运行

**测试设计原则**
- 边界值分析：最小值、最大值、边界值、空值
- 等价类划分：有效等价类、无效等价类
- 错误猜测：基于经验预测可能的错误
- 测试独立性：每个测试独立运行，互不依赖
- 测试可重复性：多次执行结果一致
