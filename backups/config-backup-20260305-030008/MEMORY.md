## 📚 核心教训

**技术层面：**
- VMware虚拟显卡不支持CUDA/Vulkan
- 全量读取MEMORY.md浪费 → QMD精准检索
- CPU模式向量生成慢（10-15分钟），需闲时生成
- **PlantUML编码复杂 → 直接安装官方工具（10次失败换来的教训）**
- **QQ语音消息：SILK编码器必需（silk-python 0.2.8），采样率24000Hz**
- **Message tool内部bug：applyTargetToParams未调用，导致参数验证失败**
- **QMD向量生成：VMware虚拟GPU导致node-llama-cpp误判CUDA，环境变量无效**
- **Android模拟器架构限制：x86_64主机无法运行ARM模拟器（QEMU2限制）** ⭐
- **环境清理重要性：及时删除无用配置，避免空间浪费（节省4.2GB）** ⭐
- **上下文监控失效：技能只有文档无脚本 → 自建双重监控机制** ⭐⭐⭐⭐
- **定期备份价值：70MB备份 + 自动清理，防止数据丢失** ⭐⭐⭐
- **启动优化重要性：全量读取40% → 分层读取<10%，避免启动即超限** ⭐⭐⭐⭐⭐

**行为层面：**
- 填充词降低效率 → 直接行动
- 机器人风格 → 有观点、有温度、有个性
- 机械式回应 → 根据情境自然回应
- 每次都说"官家，在！" → 可以变化
- **复杂问题 → 优先使用官方工具，不要自己造轮子**

**理解层面：**
- 心跳不是为自己检查系统
- 而是为人类巡检外部世界
- 每项检查都消耗token → 精打细算
- 有价值才联系，无重要事项就安静
- **从失败中学习，坚持不懈，52分钟从需求到完成**

**语音系统层面：**
- 双系统互备：本地Whisper（离线+隐私）+ 云端硅基流动（快速+准确）

**自动化测试层面：**
